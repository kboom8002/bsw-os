## 1. 왜 "업종별" 리포트인가 — 시장 소구력 분석

### 1.1 업종별 리포트의 3가지 마켓 파워

```
┌───────────────────────────────────────────────────────────────┐
│                                                               │
│   ① 연관성(Relevance)                                        │
│   "우리 업종"의 이야기이므로 즉각적 관심 유발                 │
│                                                               │
│   ② 경쟁심리(Competitive Trigger)                            │
│   "같은 업종 경쟁사 대비 내 순위"를 알고 싶은 본능            │
│                                                               │
│   ③ 행동 유발(Action Driver)                                 │
│   업종 특화 처방이므로 즉각적 실행 가능                       │
│                                                               │
└───────────────────────────────────────────────────────────────┘
```

### 1.2 업계 선례의 성공 패턴

| 리포트 | 발행사 | 구조 | 시장 반응 |
|--------|--------|------|----------|
| **2026 AEO/GEO Benchmarks** | Conductor | 10개 GICS 산업 × 22개 하위 | 13,770 도메인 분석, 마케팅 업계 표준 레퍼런스로 자리매김 |
| **AI Search Performance** | Profound | 업종별 Answer Share 랭킹 | PR/미디어 인용 다수, 고객 획득 리드젠 핵심 자산 |
| **Magic Quadrant** | Gartner | 분야별 2×2 매트릭스 | 구매 의사결정 사실상 표준 — 리더 선정 시 매출 +15~25% |
| **Forrester Wave** | Forrester | 카테고리별 점수·순위 | 엔터프라이즈 벤더 선정 시 필수 참조 |

> **공통점:** 모든 성공한 업계 리포트는 **"특정 카테고리 내의 상대적 위치"**를 보여줍니다. BSW-OS가 28개 업종별로 이 위치를 측정할 수 있다면, 한국 시장에서 **AEO/GEO 분야의 Gartner 포지션**을 확보할 수 있습니다.

### 1.3 BSW-OS가 이것을 발표해야 하는 근거

| 요건 | BSW-OS 보유 여부 |
|------|-----------------|
| 업종별 프로브 질문 세트 | ✅ 28업종 × L1~L7 레이어 |
| 실제 AI 엔진 호출 인프라 | ✅ SearchProviderFactory (ChatGPT, Gemini, Perplexity) |
| 업종별 동적 가중치 | ✅ 5매크로 × 28업종 × 7차원 |
| 경쟁사 벤치마킹 프레임워크 | ✅ AIPR (AI Power Ranking) |
| 통합 지표 체계 | ✅ AEPI, Per-Layer, BAIR, KAIVI |
| 전략적 처방 엔진 | ✅ GapAnalyzer (4-Quadrant Prescription) |

## 3. BSW-OS의 압도적 차별점

### 3.1 Conductor에 없는 BSW-OS만의 3가지

```
┌──────────────────────────────────────────────────────────┐
│                                                          │
│  ① 업종별 동적 가중치                                   │
│     Conductor: 의료든 패션이든 동일 AI SOV 기준          │
│     BSW-OS: 의료=Authority 30%, 패션=Comparative 30%     │
│     → "같은 점수라도 업종마다 의미가 다르다"를           │
│        세계 최초로 정량화                                │
│                                                          │
│  ② 엔티티 레벨 반영도 (AEPI)                            │
│     Conductor: 도메인 단위 Citation Rate만 측정          │
│     BSW-OS: 브랜드의 핵심 엔티티(제품·서비스·전문성)     │
│     각각이 AI에 정확히 반영되는지 4단계 분류             │
│                                                          │
│  ③ 레이어별 전략 분해 (Per-Layer)                       │
│     Conductor: 전체 AI SOV만 제공                        │
│     BSW-OS: "업종 일반(IRI) vs 브랜드 방어(BDR) vs       │
│     경쟁 비교(CWR) vs 기회(OPP)"로 전략 분해            │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

## 5. 리포트 구성 체계 — 3-Layer Report Architecture

### 5.1 리포트 발행 체계

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  Layer 1: 연간 종합 리포트 (Annual)                        │
│  "2026 대한민국 업종별 AI 가시성 총람"                      │
│  → 5매크로 전체 요약 + 업종 간 비교 + 트렌드              │
│                                                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Layer 2: 분기 업종 심층 리포트 (Quarterly)                 │
│  "2026 Q3 스킨케어/뷰티 AEO 경쟁 리포트"                   │
│  → 해당 업종 Top 10~20 브랜드 랭킹 + AEPI + Per-Layer     │
│                                                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Layer 3: 속보성 인사이트 (Monthly/Event-driven)           │
│  "AI 검색에서 ○○ 업종의 판도가 바뀌고 있다"               │
│  → 특정 브랜드의 급등/급락, 신규 진입 브랜드 등            │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 6. 측정 방법론 — 프로브 설계부터 지표 산출까지

### 6.1 전체 파이프라인

```
Phase 1              Phase 2              Phase 3             Phase 4
프로브 설계          데이터 수집          지표 산출           리포트 생성
                                                             
┌──────────┐    ┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│ 업종별   │    │ Multi-Engine │    │ 지표 계산    │    │ 랭킹·차트   │
│ 질문 풀  │───▶│ 동시 호출    │───▶│ BAIR·AEPI   │───▶│ 시계열 비교 │
│ × 브랜드 │    │ ×N회 반복    │    │ Per-Layer    │    │ 4사분면     │
│ × 경쟁사 │    │              │    │ KAIVI        │    │ 레이더 차트 │
└──────────┘    └──────────────┘    └──────────────┘    └──────────────┘
```

### 6.2 Phase 1: 프로브 설계 — 업종별 Fair Probe Set

각 업종 리포트에 사용할 프로브 질문 세트를 공정하게 설계합니다.

| 레이어 | 질문 수 | 목적 | 예시 (스킨케어) |
|--------|--------|------|---------------|
| **L1** (업종 공통) | 10~15 | 업종 일반 질의 | "민감성 피부 스킨케어 추천해줘" |
| **L2** (경쟁 비교) | 브랜드수×3 | 1:1 비교 | "○○ vs △△ 어디가 더 좋아?" |
| **L3** (성분/기술) | 5~10 | 전문 질의 | "세라마이드 함유 보습제 추천" |
| **L5** (YMYL) | 3~5 | 고신뢰 질의 | "아토피 피부에 안전한 스킨케어는?" |
| **L6** (트렌드) | 3~5 | 시의성 질의 | "2026 스킨케어 트렌드 브랜드는?" |
| **L7** (브랜드 직접) | 브랜드수×2 | 브랜드 방어 | "○○ 브랜드 대표 제품 알려줘" |

**Fair Probe 원칙:**
- L2(비교): 모든 브랜드 조합이 동일 횟수 등장 (round-robin)
- L7(브랜드): 모든 브랜드에 동일 유형 질문 적용
- 질문 순서: 무작위 셔플

### 6.3 Phase 2: 데이터 수집 — Multi-Engine × N회 반복

| 항목 | 설정 |
|------|------|
| **AI 엔진** | ChatGPT (Search), Gemini (Grounding), Perplexity |
| **반복 횟수** | 엔진당 최소 3회 (통계적 안정성) |
| **수집 항목** | 응답 전문, 인용(Citation) 배열, 인용 위치(inline/footer) |
| **수집 주기** | 분기 1회 (정식 리포트) + 월간 스냅샷 (트렌드 추적) |

### 6.4 Phase 3: 지표 산출

수집된 데이터에서 4대 핵심 지표를 산출합니다.

| 지표 | 산출 공식 | 단위 |
|------|----------|------|
| **BAIR** | 0.35·BSF + 0.25·AAS_w + 0.20·OCR + 0.10·SWEL + 0.10·MQ | 0~100 |
| **AEPI** | 7차원 가중합 × techMod × eeatMod × macroMul (업종별 동적 가중치) | 0~100 |
| **IRI** | (일반 질문 중 브랜드 언급 수) / 전체 × 100 | 0~100 |
| **BDR** | (L7 중 자사 브랜드 답변 수) / 전체 L7 × 100 | 0~100 |
| **CWR** | (L2 중 자사 1위 수) / 전체 L2 × 100 | 0~100 |
| **OPP** | (일반 질문 중 무브랜드 답변 수) / 전체 × 100 | 0~100 |

### 6.5 Phase 4: 리포트 생성

- 전분기 데이터와 비교하여 순위 변동 산출
- 4사분면 매트릭스(BDR×CWR) 포지셔닝
- 7차원 레이더 차트 (업종 평균 vs Top 브랜드)
- GapAnalyzer 기반 업종별 맞춤 처방

## 7. 신뢰성 확보 전략 — 대외 발표에 필요한 8가지 장치

### 7.1 방법론 투명성

| # | 장치 | 내용 |
|---|------|------|
| 1 | **Methodology Appendix** | 모든 리포트에 측정 방법론 부록 첨부 — 프로브 설계, 엔진 목록, 반복 횟수, 공식 |
| 2 | **Proxy Caveat** | "본 리포트는 설계된 프로브 세트 기반의 대리(Proxy) 측정입니다" 의무 명시 |
| 3 | **표본 크기 공개** | 업종당 총 프로브 수, 브랜드 수, 응답 수를 명시 |

### 7.2 편향 방지

| # | 장치 | 내용 |
|---|------|------|
| 4 | **Fair Probe Certification** | 모든 브랜드에 동일 조건 적용됨을 증명하는 설계 원칙 공개 |
| 5 | **Multi-Engine Consensus** | 단일 엔진이 아닌 3+ 엔진의 합의(Consensus) 기반 측정 |
| 6 | **isEstimated 투명성** | 프로브 데이터 부족 브랜드는 "추정치" 표시 — 과장 방지 |

### 7.3 외부 검증

| # | 장치 | 내용 |
|---|------|------|
| 7 | **학술 인용** | IR 이론(Jaccard, NDCG, Weighted Relevance) 기반임을 명시 |
| 8 | **재현 가능성 선언** | "동일 프로브 세트와 동일 시점에서 동일 결과를 얻을 수 있습니다" |

## 8. 발표 형태 및 채널별 전략

### 8.1 발표 형태 매트릭스

| 형태 | 주기 | 타겟 | 포맷 | 기대 효과 |

| **분기 업종 심층** | 분기 1회 × 5~10업종 | 업종 담당 마케터 | PDF + 인터랙티브 웹 | 리드젠, 고객 획득 |
| **월간 인사이트** | 월 1회 | 기존 고객 + 잠재 | 블로그 + 뉴스레터 | 리텐션, SEO |
| **실시간 대시보드** | 상시 | BSW-OS 고객 | SaaS 대시보드 | 프리미엄 기능, 업셀 |

### 8.2 채널별 콘텐츠 전략

#### 📰 보도자료 / PR
```
헤드라인: "2026 Q3 스킨케어 업종 AI 가시성 1위 브랜드는 ○○"
서브: "BSW-OS 분석, AI 검색에서 브랜드 방어율(BDR) 업종 평균 42% — 
      절반 이상의 브랜드가 AI에서 자사 이름도 지키지 못하고 있어"
```

> **PR 핵심:** 업종 평균 수치 + 충격적 인사이트 → 미디어 인용 유도

#### 📊 LinkedIn / 업계 매체
- 4사분면 매트릭스 이미지 1장 + 3줄 인사이트
- "귀사는 어느 사분면에 위치하나요?" → CTA

#### 🎤 컨퍼런스 / 세미나
- "업종별 AI 가시성 현황" 발표 → 사례 기반
- 실시간 데모: "지금 이 자리에서 귀사 브랜드를 측정해 드립니다"

#### 💻 Gated Content (리드젠)
- 전체 리포트는 이메일 등록 후 다운로드
