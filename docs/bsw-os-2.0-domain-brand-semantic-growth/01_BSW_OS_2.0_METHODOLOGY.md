# 1. 문제정의

전통적 SEO 운영은 다음 흐름을 전제로 한다.

```text
키워드 조사
→ 콘텐츠 제작
→ 검색순위
→ 트래픽
```

AI 검색·대화형 검색·에이전트 시대에는 Keyword만으로 충분하지 않다. 사용자의 실제 요청은 다음 요소를 동시에 포함한다.

```text
누가
어떤 상황에서
어떤 제약을 가지고
어떤 판단을 내리기 위해
어떤 형태의 답을 원하는가
```

따라서 운영대상은 다음으로 확장돼야 한다.

```text
Question
+ Customer Situation
+ Intent
+ Entity
+ Concept
+ Evidence
+ Brand Authority
+ Answer Asset
+ Outcome
```

# 2. 정본 정의

> **BSW-OS 2.0은 우산 도메인의 질문·검색수요·AI 답변수요·개념·경쟁·지식공백을 지속적으로 감지하고, 이를 Domain TCO와 Brand SSoT·Evidence에 연결하여, 적합한 브랜드와 자산 유형에 전략적으로 배분하고, SEO·AEO·GEO 성과로 학습하는 Domain–Brand Semantic Growth OS다.**

BSW-OS는 다음을 수행한다.

1. 무엇이 질문되고 있는지 발견한다.
2. 질문이 어떤 상황과 의사결정에 속하는지 해석한다.
3. 어떤 개념과 근거가 필요한지 정의한다.
4. 어떤 브랜드가 답할 자격과 능력이 있는지 평가한다.
5. 기사·FAQ·Answer Card·Place·Product·Structured Data 중 적합한 자산을 선택한다.
6. Article Work OS 또는 다른 Work OS에 Work Order를 보낸다.
7. 승인 결과물을 Domain Hub·Brand Site·AI홈피·채널에 투영한다.
8. Search·AI Answer·고객행동 결과로 질문지도와 전략을 갱신한다.

# 5. Question-centered Semantic Growth

BSW-OS의 기본 단위는 Keyword가 아니라 Canonical Question이다.

```text
Question
+ Who
+ Where
+ When
+ Constraint
+ Desired Outcome
+ Anxiety
+ Decision Stage
= Customer Situation
```

질문은 다음 Graph의 중심이다.

```text
Question
↔ Customer Situation
↔ Intent
↔ Entity
↔ TCO Concept
↔ Evidence Requirement
↔ Brand
↔ Existing Asset
↔ Gap
↔ Outcome
```

질문관계는 `parent`, `child`, `follow_up`, `prerequisite`, `comparison`, `alternative`, `condition_variant`, `contradiction`, `risk_variant`, `seasonal_variant`, `localization_variant`를 지원한다.

# 6. TCO와 Semantic Boundary

TCO는 용어집이 아니다.

```text
TCO
= Concept
+ Boundary
+ Relation
+ Competency Question
+ Evidence Requirement
+ Risk
+ Decision Use
```

스킨케어 핵심 경계:

```text
Cosmetic Benefit ≠ Medical Treatment
Ingredient Function ≠ Product-level Effect
User Review ≠ Clinical Evidence
Hydration ≠ Barrier Repair
Sensitive Skin ≠ Diagnosed Skin Disease
Suitable For ≠ Guaranteed Result
```

제주 핵심 경계:

```text
Place Location ≠ Travel Area
Scenic View ≠ Guaranteed Visibility
Opening Hours ≠ Real-time Availability
Local ≠ Locally Produced
Accessible ≠ Fully Barrier-free
Review Statement ≠ Verified Fact
```

# 8. Opportunity Method

Opportunity는 검색량 하나가 아니다.

```text

=

× Coverage Gap
× Answerability
× Evidence Readiness
```

이는 예측식이 아니라 평가축을 누락하지 않기 위한 개념적 구성이다.

# 9. Domain Hub와 Brand Surface

Domain Hub는 일반지식·선택기준·중립비교·질문지도·데이터·트렌드를 소유한다.

Brand Surface는 공식 브랜드정보·제품·서비스·이용조건·자체 Evidence·FAQ·예약·구매·상담을 소유한다.

```text
Hub
= 상위개념·선택기준·중립정보

Brand
= 자사 사실·조건·구체적 해결
```

동일문서를 복제하지 않고 승인 Knowledge에서 Owner·Audience별 Projection을 생성한다.

# 14. Autonomous Media 정의

> **Domain Brand Autonomous Media System은 승인된 Domain Pack·Source·Profile·Risk Boundary 안에서 반복 운영을 자동화하고, 예외·고위험 Claim·출시책임을 사람에게 Escalation하는 폐루프형 미디어 운영체계다.**

90% 자동화는 Signal 수집, 중복제거, 질문군집, 전략후보, Work Order, 자료정리, 초안, Validator, Projection, 측정, Refresh의 업무자동화를 뜻한다. 자동발행률 90%를 뜻하지 않는다.

# 16. 비주장

현재 입증되지 않은 주장:

- 검색순위 보장
- 특정 LLM 인용 보장
- Pattern의 예측력
- 90% 자동화의 매출보장
- AI Answer Observation의 대표성

성과는 `OBSERVED`, `ASSOCIATED`, `EXPERIMENTALLY_SUPPORTED`, `CAUSAL_NOT_CLAIMED`로 관리한다.
