# BSW-OS 브랜드 진단/평가 전략 및 가치제안

## 2중 다이아몬드(Double Diamond) 사고 프레임워크 적용

## 💎 Diamond 1: 올바른 문제 정의 (Discover → Define)

### 1.1 Discover — 현실 발산 탐색

#### 소상공인/자영업자의 디지털 현실

| 현실 | 데이터 포인트 | 영향 |
|------|-------------|------|
| **Near-Zero 디지털 자산** | 웹사이트 보유율 < 20%, SEO 인식 < 5% | AI 검색 시대에 완전 소외 |
| **AI 검색 블랙홀** | ChatGPT/Gemini에서 소상공인 브랜드 인용률 < 1% | 디지털 존재감 없음 |
| **비용 장벽** | 웹사이트 제작 200~500만원, SEO 컨설팅 월 100만원+ | 투자 불가 |
| **지식 장벽** | Schema.org, E-E-A-T, AEO 개념 자체 모름 | 스스로 개선 불가 |
| **시간 장벽** | 일일 12시간+ 영업, 콘텐츠 제작 여력 없음 | 디지털 방치 |
| **측정 불가** | 자신의 디지털 수준이 업계 어느 위치인지 모름 | 방향성 상실 |

#### 중소기업의 디지털 현실

| 현실 | 상세 | 영향 |
|------|------|------|
| **레거시 웹사이트** | 5~10년 전 제작, 모바일 미대응, Schema 없음 | AI 크롤러에 무시됨 |
| **콘텐츠 사일로** | 블로그·SNS·웹사이트 분산, 통합 전략 없음 | 시너지 제로 |
| **경쟁사 대비 모호** | "우리가 잘하는 건지, 못하는 건지 모름" | 개선 우선순위 불명 |
| **담당자 역량 한계** | 마케팅 1~2인, SEO 전문가 없음 | 외주 의존 |

#### 지역 공공기관의 디지털 현실

| 현실 | 상세 | 영향 |
|------|------|------|
| **형식적 웹사이트** | 법적 요건만 충족, 시민 체감 가치 낮음 | AI 답변에 미반영 |
| **지역 브랜딩 부재** | "제주 동쪽" "성수동" 등 지역 아이덴티티 미구축 | Place Brand 경쟁력 없음 |
| **관광·상권 정보 파편화** | 네이버 플레이스, 인스타, 공공데이터 산재 | 통합 답변 불가 |
| **성과 측정 부재** | 디지털 마케팅 ROI 측정 체계 없음 | 예산 확보 어려움 |

### 1.2 Define — 핵심 문제 수렴 정의

> **핵심 문제 정의 (HMW: How Might We):**
>
> 🎯 *"디지털 자산이 전무(near-zero)하거나 열악한 소상공인·중소기업·지역공공기관이,  
> (1) 자신의 현재 수준을 업종·지역·테마 벤치마크 대비 정확히 파악하고,  
> (2) AI 검색 시대에 필요한 최소 품질 이상의 디지털 자산을 즉시 구축하며,  
> (3) 지속적으로 경쟁력을 자동 개선할 수 있게 하려면 어떻게 해야 하는가?"*

#### 문제의 3계층 구조

```
┌─────────────────────────────────────────────┐
│  L1. 인식 문제: "나는 어디에 있는가?"          │  ← BSW-OS 진단/벤치마크
├─────────────────────────────────────────────┤
│  L2. 구축 문제: "어떻게 시작하는가?"           │  ← GENESIS 즉시 개설
├─────────────────────────────────────────────┤
│  L3. 성장 문제: "어떻게 계속 좋아지는가?"      │  ← QIS + Growth Orchestrator
└─────────────────────────────────────────────┘
```

**과금 모델:** Free (리드 생성) → Basic (상세 보고서) → Pro (처방전)

#### VP-4. 📍 "지역 브랜드 통합 진단" (Place Brand Assessment)

| 진단 축 | BSW-OS 기능 | 출력 |
|---------|------------|------|

| **Industry 축** | 해당 지역 업종별 배치 벤치마크 | 업종별 경쟁력 맵 |
| **Theme 축** | Vortex 테마 매핑 (17개 카테고리) | 테마 콘텐츠 갭 |
| **통합** | classifySignalAxis → cross_axis 분석 | 3축 교차 기회 발견 |

**활용 예시:**
```
"제주 동쪽 카페" 검색 시 AI 답변 분석:
  → 현재 인용되는 브랜드 3개 식별
  → 미인용 브랜드 42개 식별 (갭)
  → Place Hub 콘텐츠 미션 15건 자동 발행
  → 3개월 후 AI 인용률 23% → 67% 상승
```

#### VP-5. 🔄 "자동 성장 엔진" (Continuous Growth Engine)

**대상:** 기존 웹사이트 보유 중소기업  
**BSW-OS QIS + Growth Orchestrator 연계:**

```
일일 자동 사이클:
  
  KST 12:00 ─ BSW Cron: Hub에서 신호 Pull → 예측 질문 생성
       │
       ├─ 3축 라우팅: "제주 맛집 주차" → place_slug: jeju
       │
       ├─ Hub Push: 예측 질문 + 축 정보 + 추천 포맷
       │
  KST 13:00 ─ Hub: 테넌트에게 콘텐츠 미션 발행
       │
  KST ~  ─ 테넌트/AI: 콘텐츠 작성 (answer_card, case_study)
       │
  KST 21:00 ─ Hub Cron: 출현 피드백 → BSW
       │
       └─ BSW: 예측 모델 재보정 (accuracy_tracker)
```

**핵심 KPI:**
- 월간 예측 질문 공급: 50~200건/테넌트
- AI 인용률 목표 증가: +5~15%/분기
- 콘텐츠 미션 완료율: 40~60%
