## 1. 개요 및 측정 목표

### 1.1 무엇을 측정하는가?

본 벤치마크는 **AI 검색 엔진(ChatGPT Search, Gemini Grounding)이 특정 브랜드를 얼마나 자발적으로 언급하고 인용하는지**를 정량화하는 시스템입니다. 전통적 SEO가 "구글 결과 1페이지 노출"을 목표로 했다면, AEO(Answer Engine Optimization)는 AI가 생성하는 답변 속에 브랜드가 포함되는지를 측정합니다.

### 1.2 핵심 지표

| 지표 | 영문명 | 정의 | 측정 방식 |
|------|--------|------|-----------|
| **AAS** | AI Answer Share | AI 답변에서 브랜드가 언급된 비율 | 질문 응답 텍스트 내 키워드 매칭 |
| **OCR** | Observed Citation Rate | AI가 공식 도메인을 출처로 인용한 비율 | 응답 내 URL 도메인 매칭 |
| **BAIR** | Brand AI Index Rank | AAS·OCR·점유 순위 종합 지수 | 가중 복합 산식 |

## 2. 질문 세트 설계의 과학적 근거

### 2.1 왜 질문 세트가 중요한가?

AI 검색 엔진은 **질문(쿼리)에 따라 다른 정보를 인출**합니다. 하나의 질문만으로 브랜드 가시성을 평가하면 다음 문제가 생깁니다.

- **과적합(Overfitting)**: "더청담스튜디오 특징은?"처럼 브랜드명을 직접 포함한 질문만 쓰면 AAS가 인위적으로 높아짐
- **과소평가(Underfitting)**: "웨딩 사진 예약 방법은?"처럼 너무 일반적인 질문만 쓰면 특정 브랜드 언급 빈도가 낮아짐
- **표본 편향(Sampling Bias)**: 한 유형의 질문만 사용하면 특정 단계(인지/고려/결정)에 강한 브랜드가 과대 측정됨

**해결책**: 소비자 정보 탐색 여정 전체를 커버하는 **다층 레이어(Multi-Layer) 질문 풀(Pool)** 설계

### 2.2 골디락스 원칙(Goldilocks Principle)

패널명에 "Goldilocks"가 포함된 이유는, 각 질문이 다음 세 가지 조건을 만족해야 하기 때문입니다.

```
너무 구체적(too specific)  →  브랜드 직접 지명 → AAS 인위 상승
너무 일반적(too generic)   →  카테고리만 탐색 → AAS 인위 하락
적정 수준(just right)      →  실제 소비자 쿼리와 유사 → 현실 반영
```

골디락스 질문의 예:
- ❌ 너무 구체적: "더청담스튜디오 청담동 위치 몇 번지인가요?"
- ❌ 너무 일반적: "결혼은 어떻게 하나요?"
- ✅ 적정: "청담 웨딩 스튜디오 가격대 비교"

## 3. 레이어 구조 설계

### 3.1 7단계 레이어 아키텍처

본 패널은 소비자 정보 탐색 여정(Customer Information Journey)과 AI 검색 엔진의 응답 패턴을 동시에 반영한 **7개 레이어 80개 질문**으로 구성됩니다.

```
┌─────────────────────────────────────────────────────────────┐
│  L1  Universal (15Q)   — 카테고리 기초 지식 탐색              │
│  L2  Competitive (15Q) — 브랜드 비교·경쟁 구도 탐색          │
│  L3  Ingredient (10Q)  — 촬영 기술·스타일 심화 탐색          │
│  L4  Journey (10Q)     — 고객 여정 단계별 탐색               │
│  L5  YMYL Safety (10Q) — 법적·안전·개인정보 이슈 탐색        │
│  L6  Trend (10Q)       — 2026 최신 트렌드 탐색               │
│  L7  Brand Override (10Q) — 특정 브랜드 전용 심화 탐색       │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 레이어별 설계 근거

#### L1. Universal — 카테고리 기초 지식 (15Q)

**과학적 근거**: 소비자의 70% 이상은 브랜드 탐색 전에 카테고리 지식을 먼저 습득합니다(ZMOT: Zero Moment of Truth 이론). AI 검색 엔진은 이런 일반 질문에 답할 때도 구체적 브랜드를 자발적으로 언급하는 경향이 있습니다.

**측정 목적**: 
- AI가 일반 카테고리 답변 중 **자발적으로** 브랜드를 언급하는지 측정
- 브랜드 권위(Authority)와 카테고리 연관성 파악

**예시 질문**: "웨딩 스냅 촬영이란 무엇이고 야외/실내 어느 게 나을까요?"

#### L2. Competitive — 경쟁 구도 비교 (15Q)

**과학적 근거**: AI 검색 엔진은 비교 질문에서 브랜드를 가장 명확하게 언급합니다. "A vs B" 또는 "추천 TOP 5"형 질문은 AI가 특정 브랜드를 직접 호명할 가능성이 가장 높습니다.

**측정 목적**:
- 경쟁 환경에서의 상대적 가시성(Relative Visibility) 측정
- 직접 비교 질문에서의 브랜드 포함률 측정

**핵심 질문**: "서울 웨딩 스튜디오 추천 TOP 5" → 5개 브랜드 전체의 AAS를 단 한 번의 질문으로 측정 가능

**설계 포인트**: L2 질문의 `must_include` 필드에 측정 대상 브랜드명을 명시하여, 질문이 실제로 브랜드를 평가하는 질문임을 검증 가능하게 구조화함

#### L3. Ingredient — 기술·스타일 심화 (10Q)

**과학적 근거**: 전문 지식 질문(예: "AI 보정 기술이 웨딩 사진에 미치는 영향")에 답할 때, AI는 해당 기술에 특화된 브랜드를 예시로 언급하는 경향이 있습니다. 이는 AI 학습 데이터 내 브랜드의 전문성 포지셔닝을 반영합니다.

**측정 목적**: 브랜드가 특정 기술/스타일과 연관 짓여 학습되었는지 측정 (Conceptual Association)

#### L4. Journey — 고객 여정 단계 (10Q)

**과학적 근거**: McKinsey의 Customer Decision Journey 모델에 따르면 소비자는 인지(Awareness) → 고려(Consideration) → 결정(Decision) → 사후(Post-purchase) 단계를 거칩니다. 각 단계별로 AI 검색 패턴이 다르며, 브랜드 언급 가능성도 다릅니다.

**설계**: 각 질문에 `decision_stage` 필드를 명시하여 어느 여정 단계의 소비자 쿼리를 시뮬레이션하는지 추적 가능

#### L5. YMYL Safety — 법적·안전 (10Q)

**과학적 근거**: Google의 YMYL(Your Money, Your Life) 기준에 따르면, 법적/재정적 영향이 있는 질문에서 AI는 더 권위 있고 신뢰할 수 있는 브랜드를 우선 인용하는 경향이 있습니다. 또한 AI 답변의 정확성 평가 측면에서, 계약·법률 관련 질문에 대한 응답 품질이 브랜드 신뢰도에 영향을 미칩니다.

**측정 목적**: 고위험 질문(High-risk query)에서 브랜드의 신뢰도 인지 여부 측정. `weight: 1.3~1.4`로 일반 질문보다 가중치를 높게 부여.

#### L6. Trend — 2026 트렌드 (10Q)

**과학적 근거**: AI 검색 엔진의 학습 데이터는 시간적 편향(Recency Bias)을 가집니다. 최신 트렌드 질문에서 어떤 브랜드가 언급되는지는 AI 훈련 데이터 내 해당 브랜드의 최근성(Freshness)을 간접적으로 반영합니다.

**측정 목적**: 브랜드의 "트렌드 적합성" 인지 여부 — AI가 최신 트렌드 답변에서 해당 브랜드를 연상하는지 측정

#### L7. Brand Override — 브랜드 전용 (10Q)

**과학적 근거**: 브랜드명을 명시적으로 포함한 쿼리에서의 AAS는 **브랜드의 AI 학습 데이터 내 정보 풍부도(Knowledge Richness)**를 직접 측정합니다. AI가 브랜드에 대해 얼마나 많은 정보를 갖고 있는지, 그 정보의 정확성은 어떤지를 파악할 수 있습니다.

**설계 포인트**: 브랜드당 2개 질문을 배정하여 단일 질문의 샘플 노이즈를 줄임. 총 10Q로 5개 브랜드 × 2회 보장.

## 4. 샘플링 전략

### 4.1 가중 샘플링(Weighted Sampling)

전체 80개 질문에서 `sampleQuestionsForLight` 수만큼 샘플링할 때, **단순 랜덤 샘플링**이 아닌 **레이어 가중 샘플링**을 적용합니다.

```
Daily 경량 측정 (20Q 샘플링) 시 가중치:
  L7 (Brand Override): 50% → 10개 (브랜드당 2개 보장)
  L2 (Competitive):    30% → 6개
  L1/L3/L4/L5/L6:     20% → 4개
```

**가중치 설계 근거**:
- **L7 우선**: 브랜드 가시성 측정의 핵심이므로 전체의 절반 보장
- **L2 차선**: 비교 질문이 AAS 측정에서 가장 정보량이 높음 (한 질문에서 여러 브랜드 동시 측정)
- **기타 레이어 소수 포함**: 측정 편향 방지를 위한 다양성 확보

### 4.2 표본 크기와 통계적 신뢰성

AAS는 비율(proportion) 지표이므로, 이항분포(Binomial Distribution)에 기반한 신뢰구간을 적용합니다.

```
95% 신뢰구간 공식:
  CI = p ± z × √(p(1-p)/n)
  여기서 z = 1.96 (95% 신뢰수준), p = AAS, n = 질문 수
```

| 표본 크기(n) | AAS=10% 시 95% CI | AAS=30% 시 95% CI | 권장 해석 |
|------------|-------------------|-------------------|-----------| 
| 10Q (구버전) | 10% ± 18.6pp | 30% ± 28.4pp | 측정값 신뢰 어려움 |
| 20Q (현재)  | 10% ± 13.1pp | 30% ± 20.1pp | 순위 판별 가능 |
| 40Q (권장)  | 10% ± 9.3pp  | 30% ± 14.2pp | 정밀 비교 가능 |
| 80Q (전체)  | 10% ± 6.6pp  | 30% ± 10.0pp | 학술 수준 정밀도 |

> **현재 20Q의 한계**: 개별 측정값보다는 **추세(Trend)와 순위(Rank)** 변화에 집중하는 것이 통계적으로 적절합니다.

## 5. 키워드 매칭 설계

### 5.1 다중 키워드 전략

단일 키워드는 AI 응답의 표기 다양성(대소문자, 띄어쓰기, 영문/한글 혼용)을 처리하지 못합니다.

```typescript
// 더청담스튜디오 키워드 예시
keywords: [
  '더청담스튜디오',    // 붙여쓰기
  '더청담',           // 축약
  '청담스튜디오',     // 부분 매칭
  'cheongdam studio', // 영문 소문자
  'chungdam studio',  // 로마자 표기 변형
  'thecheongdam',     // 도메인형
  'thechungdam',      // 도메인 변형
]
```

**설계 원칙**:
1. **한글 정체**: 공식 브랜드명 그대로 (`더청담스튜디오`)
2. **한글 축약**: 소비자가 자주 쓰는 약칭 (`더청담`)
3. **영문 표기**: AI가 영어로 응답할 때를 대비 (`cheongdam studio`)
4. **로마자 변형**: 외래어 표기 차이 (`chungdam` vs `cheongdam`)
5. **도메인형**: OCR 측정과 연동 (`thecheongdam`)

### 5.2 오탐(False Positive) 방지

일부 브랜드명은 보통명사와 충돌할 수 있습니다. 예를 들어 "섬 스튜디오"는 "섬(島)"이라는 보통명사와 결합하여 오탐 위험이 있습니다.

**해결 전략**:
- AI 모델을 정확히 사용했을 때(`gemini-2.5-flash`) 오탐이 발생하는지 실측 검증
- 오탐이 확인된 경우에만 키워드를 한정적 형태로 교체
- 오탐의 원인이 AI 모델 자체의 오류일 경우 키워드 변경보다 모델 교체가 우선

> ⚠️ **사례**: 이전 버전에서 `gemini-3.5-flash`(존재하지 않는 모델명)를 사용하여 모든 브랜드 AAS가 0%로 나오는 치명적 버그가 있었습니다. 이는 키워드 문제가 아닌 모델 호출 오류였으며, `gemini-2.5-flash`로 수정 후 정상 측정이 확인되었습니다(2026-06-04).

## 6. 질문 품질 관리 (Quality Control)

### 6.1 질문별 메타데이터 구조

각 질문에는 측정 품질을 보장하기 위한 10가지 메타데이터 필드가 정의되어 있습니다.

```typescript
interface Question {
  question_text: string;      // 실제 AI에 전달되는 쿼리
  intent_context: string;     // 질문 의도 분류
  target_keyword: string;     // 핵심 탐색 키워드
  risk_level: 'low' | 'medium' | 'high'; // YMYL 위험 수준
  decision_stage: string;     // 소비자 여정 단계
  question_type: string;      // 질문 유형
  weight: number;             // AAS 계산 시 가중치 (0.8~1.4)
  query_variants: string[];   // 동의어/변형 질문
  must_include: string[];     // AI 답변에 포함되어야 할 정보
  should_include: string[];   // AI 답변에 포함되면 좋은 정보
  must_not_do: string[];      // AI 답변에 포함되면 안 되는 내용
  layer: string;              // 레이어 분류
}
```

### 6.2 가중치(Weight) 설계

```
weight 1.4  →  YMYL 최고 위험 (계약서 필수 항목)
weight 1.3  →  법적 분쟁, 소비자 보호, 개인정보
weight 1.2  →  브랜드 전용 질문, 종합 비교
weight 1.1  →  특정 브랜드 기능 질문
weight 1.0  →  표준 정보성 질문
weight 0.9  →  참고용 일반 정보
weight 0.8  →  낮은 관여도 질문 (인화, 리뉴얼 등)
```

**가중치 적용 근거**: YMYL 질문에서의 브랜드 언급은 AI가 해당 브랜드를 '신뢰할 수 있는 정보원'으로 인식한다는 신호이므로, 더 높은 가중치를 부여하는 것이 타당합니다.

### 6.3 Must/Should/Must-Not 규칙

각 질문에 정의된 `must_include`, `should_include`, `must_not_do` 필드는 향후 **AI 답변 품질 자동 평가(LLM-as-Judge)** 시스템과 연동될 수 있도록 설계되었습니다.

| 필드 | 역할 | 활용 |
|------|------|------|
| `must_include` | 답변이 반드시 포함해야 할 정보 | 답변 완결성 평가 |
| `should_include` | 포함되면 좋은 추가 정보 | 답변 풍부도 평가 |
| `must_not_do` | 포함되면 안 되는 정보/행동 | 안전성/정확성 평가 |

## 7. 측정 파이프라인 구조

### 7.1 End-to-End 파이프라인

```
질문 풀 (80Q)
    ↓ 가중 샘플링 (20Q)
    ↓
AI 쿼리 실행
    ├── ChatGPT Search (openai-search)
    └── Gemini Grounding (gemini-2.5-flash)
    ↓
응답 텍스트 수집
    ↓
키워드 매칭 (AAS 계산)
    ├── 브랜드 키워드 포함 여부 → AAS 히트
    └── 공식 도메인 URL 포함 여부 → OCR 히트
    ↓
가중 평균 산출
    ↓
Supabase 저장 (industry_benchmark_snapshots)
    ↓
리더보드 표시
```

### 7.2 AAS 계산 공식

```
AAS(brand) = (키워드 매칭 질문 수 × 해당 질문 weight) 
             / (전체 질문 수 × 평균 weight)
             × 100
```

단순 비율이 아닌 **가중 비율**을 사용하여, YMYL 고위험 질문에서의 브랜드 언급이 일반 질문보다 더 높은 가치를 갖도록 설계하였습니다.

## 8. 측정의 한계와 개선 방향

### 8.1 현재 한계

| 한계 | 원인 | 영향 |
|------|------|------|
| 높은 분산 | 20Q 소표본 | 측정값 신뢰구간 ±13pp 이상 |
| AI 응답 비결정성 | 생성형 AI의 확률적 특성 | 동일 질문도 매 측정마다 다른 답변 |
| 측정 시점 편향 | 일회성 스냅샷 | 특정 시점의 AI 상태만 반영 |
| 키워드 매칭 한계 | 단순 텍스트 비교 | 맥락 없는 언급(부정 언급 등)도 히트로 계산 |

### 8.2 개선 로드맵

```
단기 (1~3개월):
  □ 샘플 수 40Q로 확대 (신뢰구간 ±9pp로 개선)
  □ 부정 문맥 필터링 ("~는 비추천"도 히트로 잡히는 문제)
  □ 복수 측정 후 평균화 (동일 질문 3회 반복 측정)

중기 (3~6개월):
  □ LLM-as-Judge 품질 평가 통합
  □ Sentiment 분류 (긍정/중립/부정 언급 분리)
  □ 질문 변형(query_variants) 랜덤 교체

장기 (6개월+):
  □ 자동화 업종 확장 (스킨케어, 편의점 등)
  □ 다국어 측정 (영어 쿼리 AAS 별도 측정)
  □ 경쟁사 AI 엔진 추가 (Perplexity, Claude 등)
```

## 9. 참조 개념

| 개념 | 정의 | 본 시스템에서의 적용 |
|------|------|---------------------|
| ZMOT | Zero Moment of Truth (Google, 2011) — 소비자가 구매 전 온라인에서 정보를 탐색하는 순간 | L1 레이어 설계 근거 |
| AEO | Answer Engine Optimization — AI 검색 엔진의 답변에 브랜드/콘텐츠가 포함되도록 최적화 | 전체 벤치마크 목표 |
| GEO | Generative Engine Optimization — 생성형 AI가 콘텐츠를 인용하도록 최적화 | OCR 지표 설계 근거 |
| YMYL | Your Money, Your Life (Google 품질 평가 기준) | L5 레이어 가중치 설계 근거 |
| Goldilocks Principle | 너무 구체적이지도, 너무 일반적이지도 않은 최적 질문 설계 | 전체 질문 선별 원칙 |
| Binomial CI | 이항분포 기반 신뢰구간 — 비율 지표의 통계적 불확실성 계량 | 표본 크기 결정 근거 |
