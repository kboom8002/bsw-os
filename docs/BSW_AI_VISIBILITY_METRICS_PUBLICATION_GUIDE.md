## 1. 서론: 왜 AI 가시성 지표가 필요한가

### 1.1 패러다임 전환

전통적 검색 최적화(SEO)에서 AI 검색 최적화(AEO/GEO)로 전환되면서, 브랜드의 디지털 가시성을 측정하는 방식이 근본적으로 변화하고 있습니다.

| 차원 | SEO 시대 | AI 검색 시대 |
|------|---------|------------|
| **노출 형태** | 검색 결과 페이지 내 링크 순위 | AI 생성 답변 내 직접 언급·인용 |
| **사용자 행동** | 클릭 → 사이트 방문 → 탐색 | AI 답변 소비 → 선택적 인용 클릭 |
| **브랜드 인지** | 검색 순위 내 노출 횟수 | AI가 브랜드를 얼마나 정확히 추천하는가 |
| **측정 대상** | 키워드 순위, CTR, 유입량 | Answer Share, Citation Rate, Entity Fidelity |
| **경쟁 구도** | 10개 링크 경쟁 | AI가 선택한 1~3개 브랜드 경쟁 |

### 1.2 BSW-OS의 접근

BSW-OS는 이 패러다임 전환에 대응하여, **"AI가 우리 브랜드를 얼마나 정확하고 호의적으로 말하고 있는가?"**를 다차원으로 측정하는 통합 지표 프레임워크를 구축했습니다.

핵심 설계 원칙:

| 원칙 | 설명 |
|------|------|
| **실측 기반 (Measurement-First)** | 실제 AI 엔진(ChatGPT, Gemini 등)에 프로브 질문을 투입하고 응답을 분석합니다 |
| **업종 특화 (Industry-Adaptive)** | 28개 세부업종별 동적 가중치로 업종 특성을 반영합니다 |
| **다차원 분해 (Multi-Dimensional)** | 단일 점수가 아닌, 가시성·감성·인용·구조·품질의 독립 차원을 측정합니다 |
| **투명성 (Transparency)** | 모든 지표에 `isEstimated` 플래그와 데이터 신뢰도를 명시합니다 |

BSW-OS의 4대 핵심 지표는 3개 계층에 걸쳐 상호 보완적으로 설계되어 있습니다.

```

│                                                             │
│               KAIVI  (Korea AI Visibility Index)            │
│           ── 국가 수준 AI 가시성 종합 지수 ──                │
│     0.30×BAIR + 0.30×AEPI + 0.25×PerLayer + 0.15×MRI      │
│                                                             │
├─────────────────┬───────────────────────────────────────────┤
│                 │                                           │
│     BAIR        │           Per-Layer Metrics               │
│  Brand AI       │     IRI · BDR · CWR · OPP                │
│  Reputation     │     ── 전략적 전투력 진단 ──              │
│  Index          │                                           │
│                 │                                           │
├─────────────────┴───────────────────────────────────────────┤
│                                                             │
│                          AEPI                               │
│            AI Entity Presence Index                         │
│      ── 7차원 × 28업종 엔티티 반영도 ──                     │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

| 계층 | 지표 | 역할 |
|------|------|------|
| **기반층** | AEPI | 엔티티 레벨 — AI가 브랜드 정보를 얼마나 정확히 알고 있는가 (Input-side) |
| **전략층** | Per-Layer + BAIR | 성과 레벨 — AI가 브랜드를 어떻게 말하는가 (Output-side) |
| **종합층** | KAIVI | 국가 수준 — 4개 원천의 다원 복합 종합 |

## 3. AEPI — AI Entity Presence Index

> **"AI가 당신의 브랜드 핵심 정보를 얼마나 정확하게 알고 있는가"**

### 3.1 정의

AEPI(AI Entity Presence Index)는 AI 검색 엔진이 브랜드의 **핵심 엔티티(제품, 서비스, 전문성, 위치 정보 등)**를 응답에 얼마나 정확하게 반영하고 있는지를 측정하는 지수입니다.

- **범위:** 0~100
- **단위:** 점
- **주기:** 주간 또는 수시 (프로브 실행 시)

### 3.2 측정 방법론

#### 3.2.1 2단계 동적 가중치 엔진

AEPI는 업종의 비즈니스 특성을 정밀하게 반영하기 위해 2단계 동적 가중치 시스템을 적용합니다.

**Layer 1 — BM 매크로 5영역 가중치:**

업종의 비즈니스 모델(BM)에 따라 5개 핵심 영역의 상대적 중요도를 설정합니다.

| 매크로 카테고리 | Entity (e) | Content (c) | Technical (t) | Schema (s) | E-E-A-T (r) |
|----------------|-----------|-------------|---------------|-----------|-------------|
| 이커머스/D2C | 0.20 | 0.20 | 0.25 | 0.25 | 0.10 |
| 지역 서비스 | 0.15 | 0.15 | 0.20 | 0.20 | **0.30** |
| 고신뢰 전문직(YMYL) | **0.25** | 0.15 | 0.15 | 0.15 | **0.30** |
| B2B 테크/SaaS | 0.20 | **0.35** | 0.20 | 0.15 | 0.10 |
| 미디어/콘텐츠 | **0.25** | **0.30** | 0.10 | 0.15 | 0.20 |

> **설계 근거:** 의료·법률(YMYL) 업종은 E-E-A-T(경험·전문성·권위·신뢰) 가중치가 30%로 높고, 이커머스는 기술 인프라와 구조화 데이터 가중치가 높습니다. 이는 Google의 Search Quality Rater Guidelines 및 AI 검색 엔진의 정보 신뢰도 평가 방식에 기반합니다.

**Layer 2 — 세부업종별 7차원 서피스 가중치:**

28개 세부업종 각각에 대해 7개 독립 차원의 가중치를 설정합니다.

| 차원 | 의미 | 대표 프로브 유형 |
|------|------|----------------|
| **Factoid** | 사실 기반 질의 대응력 | "○○의 가격은?" "○○ 성분은?" |
| **Procedural** | 절차·방법 질의 대응력 | "○○ 사용법은?" "○○ 예약 절차는?" |
| **Comparative** | 비교·추천 질의 대응력 | "○○ vs △△ 차이는?" "최고의 ○○?" |
| **Authority** | 권위·신뢰 신호 | 전문가 인증, 수상 경력, 논문 인용 |
| **Schema.org** | 구조화 데이터 커버리지 | JSON-LD, FAQPage, Product, LocalBusiness |
| **Topical Cluster** | 토픽 클러스터 구성도 | 주제별 컨텐츠 허브 연결 강도 |
| **Local/Geo** | 지역·지리 최적화 | 위치, 영업시간, 서비스 지역 |

**예시 — 업종별 가중치 차이:**

| 차원 | 스킨케어(D2C) | 웨딩(지역) | 법률(YMYL) | IT/SaaS(B2B) |
|------|-------------|----------|----------|-------------|
| Factoid | 0.20 | 0.10 | 0.15 | 0.15 |
| Comparative | **0.25** | 0.15 | 0.10 | 0.20 |
| Authority | 0.15 | 0.10 | **0.30** | 0.10 |
| Local/Geo | 0.05 | **0.30** | 0.15 | 0.05 |

> **해석:** 스킨케어는 비교 질의(Comparative)가 중요하고, 웨딩은 지역 최적화(Local/Geo)가, 법률은 권위(Authority)가, SaaS는 비교+토픽 클러스터가 핵심입니다.

#### 3.2.2 엔티티 반영도 측정 프로세스

```
Surface Entities          AI Probe Pipeline           Reflection Analysis
(브랜드 등록 엔티티)      (실제 AI 엔진 호출)         (반영도 분류)
                                                      
┌──────────────┐     ┌──────────────────┐     ┌──────────────────────┐
│ 제품 정보    │     │ ChatGPT Search   │     │ Exact    (100%)      │
│ 서비스 정보  │────▶│ Gemini Grounding │────▶│ Partial  ( 60%)      │
│ 전문가 프로필│     │ Perplexity       │     │ Distorted( 20%)      │
│ 위치 정보    │     │ (다수 엔진 동시) │     │ Absent   (  0%)      │
│ 스키마 데이터│     └──────────────────┘     └──────────────────────┘
└──────────────┘                                       │
                                                       ▼
                                               ┌──────────────────┐
                                               │ 7차원 ERR 집계   │
                                               │ × 업종별 가중치  │
                                               │ × 매크로 보정    │
                                               │ = AEPI (0~100)   │
                                               └──────────────────┘
```

**반영도 4단계 분류:**

| 등급 | 가중치 | 판정 기준 |
|------|--------|----------|
| **Exact** (정확 반영) | 1.0 | 엔티티명 일치 + 키워드 오버랩 ≥ 80% |
| **Partial** (부분 반영) | 0.6 | 엔티티명 일치 + 키워드 오버랩 ≥ 40%, 또는 도메인 인용 |
| **Distorted** (왜곡 반영) | 0.2 | 키워드 오버랩 ≥ 20%이나 핵심 정보 변질 |
| **Absent** (미반영) | 0.0 | 엔티티 정보 부재 |

#### 3.2.3 최종 AEPI 공식

$$\text{AEPI} = \text{baseScore} \times \text{techModifier} \times \text{eeatModifier} \times \text{macroMultiplier}$$

$$\text{baseScore} = \sum_{d=1}^{7} \text{dimScore}_d \times w_d$$

- $\text{techModifier} = 0.8 + 0.2 \times \frac{\text{techScore}}{100}$ — 기술 인프라 보정
- $\text{eeatModifier} = 0.8 + 0.2 \times \frac{\text{eeatScore}}{100}$ — E-E-A-T 보정
- $\text{macroMultiplier} \in [0.90, 1.10]$ — 매크로 카테고리 보정

### 3.3 과학성 분석

| 평가 기준 | 등급 | 근거 |
|----------|------|------|
| **실측 기반** | ⭐⭐⭐⭐⭐ | 실제 AI 엔진에 프로브 질문 투입 → 응답 분석 (mock 아님) |
| **재현 가능성** | ⭐⭐⭐⭐⭐ | 동일 프로브 세트 + 동일 엔진 → 결정적 결과 (알고리즘 내 랜덤 요소 없음) |
| **이론적 기반** | ⭐⭐⭐⭐ | 정보 검색(IR) 분야의 가중 적합성(Weighted Relevance), 엔티티 인식(NER), Google E-E-A-T |
| **업종 특화** | ⭐⭐⭐⭐⭐ | 5매크로 × 28세부업종 × 7차원 — 업계 유일 수준의 정밀 커스터마이징 |
| **독창성** | ⭐⭐⭐⭐⭐ | AI 응답 내 엔티티 반영도를 체계적으로 측정하는 공개 프레임워크는 현재 없음 |

### 3.4 활용 가치

> **경영진 요약:** "우리 브랜드의 핵심 정보 30개 중 AI가 정확히 반영하는 것이 18개(AEPI=60)이며, 스키마 구조화 차원이 가장 취약합니다."

- **현상 진단:** 어떤 차원(사실형/비교형/권위 등)이 AI에서 잘 반영되고, 어떤 차원이 누락되는지 식별
- **액션 연결:** 취약 차원별 구체적 개선 처방(스키마 추가, E-E-A-T 강화, 콘텐츠 생성 등) 자동 도출
- **경쟁 비교:** 동일 프로브에서 경쟁사 엔티티 반영 여부 동시 측정 (`competitor_mentioned` 필드)

## 4. Per-Layer Metrics — 4대 전략 지표

> **"AI 전장에서 우리 브랜드의 실전 전투력을 4개 각도에서 진단한다"**

### 4.1 정의

Per-Layer Metrics는 프로브 질문의 **의도 레이어(Intent Layer)**별로 브랜드의 AI 검색 성과를 분리 측정하는 4개 전략 지표군입니다.

| 지표 | 정식 명칭 | 범위 | 한줄 정의 |
|------|---------|------|----------|
| **IRI** | Industry Readiness Index | 0~100 | 업종 일반 질문에서 브랜드가 언급되는 비율 |
| **BDR** | Brand Defense Rate | 0~100 | 브랜드 직접 질문에서 브랜드가 방어되는 비율 |
| **CWR** | Competitive Win Rate | 0~100 | 경쟁 비교 질문에서 브랜드가 1위로 언급되는 비율 |
| **OPP** | Opportunity Score | 0~100 | AI에서 아직 미개척된 질문 영역의 비율 |

### 4.2 측정 방법론

#### 4.2.1 프로브 질문 레이어 분류

모든 프로브 질문은 7개 레이어(L1~L7)로 분류되며, Per-Layer Metrics는 이 레이어를 3개 전략 그룹으로 묶어 측정합니다.

```
┌────────────────────────────────────────────────────────────┐
│                    프로브 질문 레이어                       │
├──────────────────┬─────────────────────────────────────────┤
│  일반 질문 그룹  │  L1_universal  — 업종 공통 질문         │
│  (IRI, OPP)     │  L3_ingredient — 성분·기술 질문         │
│                  │  L5_ymyl       — YMYL 고신뢰 질문       │
│                  │  L6_trend      — 트렌드·시의성 질문     │
├──────────────────┼─────────────────────────────────────────┤
│  브랜드 방어     │  L7_brand      — 브랜드 직접 질문       │
│  (BDR)          │  "○○ 브랜드의 대표 제품은?"              │
├──────────────────┼─────────────────────────────────────────┤
│  경쟁 비교       │  L2_competitive — 경쟁 비교 질문        │
│  (CWR)          │  "○○ vs △△ 어디가 더 좋아?"             │
└──────────────────┴─────────────────────────────────────────┘
```

#### 4.2.2 각 지표의 공식

**IRI (Industry Readiness Index) — 업종 준비도:**

$$\text{IRI} = \frac{\text{일반 질문 중 어떤 브랜드라도 언급된 수}}{\text{전체 일반 질문 수}} \times 100$$

> **의미:** 업종 전체의 AI 가시성 성숙도. IRI가 높으면 해당 업종에서 AI가 이미 브랜드를 적극적으로 추천하는 환경이며, 경쟁이 치열합니다.

**BDR (Brand Defense Rate) — 브랜드 방어율:**

$$\text{BDR} = \frac{\text{L7 질문 중 자사 브랜드가 언급된 수}}{\text{전체 L7 질문 수}} \times 100$$

> **의미:** "○○ 브랜드에 대해 알려줘"라는 직접 질문에서 AI가 자사 브랜드를 정확히 답하는 비율. BDR < 70이면 AI가 자사 브랜드를 제대로 인식하지 못하거나 경쟁사로 대체하고 있습니다.

**CWR (Competitive Win Rate) — 경쟁 승리율:**

$$\text{CWR} = \frac{\text{L2 질문 중 자사가 1위로 언급된 수}}{\text{전체 L2 질문 수}} \times 100$$

> **의미:** 비교·추천 질문에서 경쟁사보다 먼저(더 앞에) 언급되는 비율. 1위 판정 기준은 AI 응답 텍스트에서 자사 브랜드의 등장 위치가 경쟁사보다 앞인지 여부입니다.

**OPP (Opportunity Score) — 공략 기회 점수:**

$$\text{OPP} = \frac{\text{일반 질문 중 어떤 브랜드도 언급되지 않은 수}}{\text{전체 일반 질문 수}} \times 100$$

> **의미:** AI가 아직 특정 브랜드를 추천하지 않는 "빈 공간". OPP가 높으면 선점(First Mover) 기회가 큽니다.

#### 4.2.3 전략 매트릭스

| IRI | OPP | 해석 | 전략 |
|-----|-----|------|------|
| 높음 | 낮음 | 경쟁 과열 (Red Ocean) | CWR 향상에 집중 |
| 높음 | 높음 | (논리적 모순 — 발생 불가) | — |
| 낮음 | 높음 | 블루 오션 | 선점 콘텐츠 집중 투자 |
| 낮음 | 낮음 | 비활성 업종 | 업종 전체 AI 준비도 필요 |

### 4.3 과학성 분석

| 평가 기준 | 등급 | 근거 |
|----------|------|------|
| **정의 명확성** | ⭐⭐⭐⭐⭐ | 각 지표가 단일 비율 공식으로 정의 — 해석 모호성 제로 |
| **실측 기반** | ⭐⭐⭐⭐⭐ | 실제 AI 엔진 응답에서 직접 추출 |
| **상호 보완성** | ⭐⭐⭐⭐⭐ | IRI + OPP = 100% (완전 분해), BDR과 CWR은 독립 차원 |
| **액셔너블** | ⭐⭐⭐⭐⭐ | 각 지표가 즉각적 전략 액션과 1:1 매핑 |
| **비교 용이성** | ⭐⭐⭐⭐ | 동일 프로브 세트로 업종 내 모든 브랜드 동시 측정 |

### 4.4 활용 가치

> **경영진 요약:** "우리 브랜드 방어율(BDR)이 45%이므로, AI에 자사 이름을 물어봐도 절반 이상에서 제대로 답하지 못합니다. 반면 경쟁 승리율(CWR)은 72%로, 비교 질문에서는 강합니다."

## 5. BAIR — Brand AI Reputation Index

> **"AI가 우리 브랜드를 얼마나 호의적이고 가시적으로 말하고 있는가"**

### 5.1 정의

BAIR(Brand AI Reputation Index)는 AI 검색 응답에서 브랜드의 **가시성(Presence), 감성(Sentiment), 인용(Citation), 구조적 준비도(Structure), 언급 품질(Quality)**을 종합하는 복합 지수입니다.

- **범위:** 0~100
- **단위:** 점
- **주기:** 일간 경량 또는 주간 정밀

### 5.2 측정 방법론

#### 5.2.1 BAIR V2 공식 (가중 합산 방식)

$$\text{BAIR} = 0.35 \times \text{BSF} + 0.25 \times \text{AAS}_w + 0.20 \times \text{OCR} + 0.10 \times \text{SWEL}_m + 0.10 \times \text{MQ}$$

$$0 \leq \text{BAIR} \leq 100$$

#### 5.2.2 5개 컴포넌트 상세

| 컴포넌트 | 정식 명칭 | 가중치 | 범위 | 측정 방법 |
|----------|---------|--------|------|----------|
| **BSF** | Brand Share of Voice | **35%** | 0~100 | 프로브 응답 중 브랜드가 언급된 응답의 비율 |
| **AAS_w** | Weighted Answer Sentiment | **25%** | 0~100 | 3단계 가중 감성 분류의 평균 점수 |
| **OCR** | Observed Citation Rate | **20%** | 0~100 | AI 응답의 인용(Citation)에 브랜드 도메인이 포함된 비율 |
| **SWEL_m** | Measured Semantic Exposure Lift | **10%** | 0~100 | 시맨틱 페이지·스키마 매핑·서피스 컨트랙트 인프라 점수 |
| **MQ** | Mention Quality | **10%** | 0~100 | Strong mention(강력 추천) 비율 |

#### 5.2.3 가중치 설계 근거 (SOTA 기반)

```
Presence (BSF 35%)  ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓  ← 가시성이 최우선
Sentiment (AAS 25%) ▓▓▓▓▓▓▓▓▓▓▓▓▓       ← 긍정적 언급 품질
Citation (OCR 20%)  ▓▓▓▓▓▓▓▓▓▓           ← 트래픽 전환 동인
Structure (SWEL 10%)▓▓▓▓▓                 ← 구조적 인프라
Quality (MQ 10%)    ▓▓▓▓▓                 ← 추천 강도
```

> **BSF가 최대 가중치(35%)인 이유:** AI 검색에서 브랜드가 응답에 **등장하지 않으면** 감성·인용·품질 모두 무의미합니다. Answer Share(가시성)가 모든 후속 지표의 전제 조건이므로 최우선 가중치를 부여합니다.

> **SWEL이 실측인 이유:** 시맨틱 페이지, 구조화 데이터(Schema.org), 서피스 컨트랙트가 실제로 구축되어 있는지를 DB에서 직접 확인합니다. 하드코딩된 상수가 아닌, 인프라 투자 수준을 객관적으로 반영합니다.

#### 5.2.4 감성 분류: 3단계 가중 시스템

BSF(등장 여부)와 별개로, 브랜드가 언급될 때의 **맥락 품질**을 3단계로 분류합니다.

| 등급 | 가중치 | 판정 기준 | 예시 |
|------|--------|----------|------|
| **Strong** | 1.0 | 브랜드명 주변 80자 내 강력 추천 패턴 | "추천", "최고", "1위", "best", "top pick" |
| **Neutral** | 0.3 | 단순 언급 (긍정도 부정도 아닌) | 정보 나열, 객관적 비교 |
| **Negative** | 0.0 | 부정적 맥락 또는 미언급 | "주의", "부작용", "avoid", "concern" |

$$\text{AAS}_w = \frac{\sum \text{mention\_weight}}{\text{total\_mentions}} \times 100$$

### 5.3 과학성 분석

| 평가 기준 | 등급 | 근거 |
|----------|------|------|
| **범위 보장** | ⭐⭐⭐⭐⭐ | 가중 합산 + clamp(0, 100) — 수학적 범위 초과 불가 |
| **컴포넌트 독립성** | ⭐⭐⭐⭐ | 5개 컴포넌트가 각각 독립적 데이터 소스에서 측정 |
| **투명성** | ⭐⭐⭐⭐⭐ | 프로브 데이터 부족 시 `isEstimated=true` 명시 |
| **실측 기반** | ⭐⭐⭐⭐⭐ | SWEL 포함 모든 컴포넌트가 DB 실측 (하드코딩 상수 제로) |
| **업계 정합** | ⭐⭐⭐⭐ | BSF=Answer Share, OCR=Citation Rate — 업계 표준 용어와 직접 대응 |

### 5.4 활용 가치

> **경영진 요약:** "우리 BAIR은 62점입니다. AI 응답에서 70%의 확률로 언급되지만(BSF=70), 강력 추천을 받는 경우는 30%에 불과합니다(MQ=30). 인용율(OCR=45)을 높이면 실제 트래픽 전환이 늘어날 것입니다."

**업계 표준 지표와의 대응:**

| 업계 표준 용어 | BAIR 컴포넌트 | BSW-OS 차별점 |
|-------------|-------------|-------------|
| **Answer Share** | BSF | 동일 — 프로브 기반 실측 |
| **Citation Rate** | OCR | **inline/footer 인용 위치까지 구분** |
| **Brand Sentiment** | AAS_w | **3단계 가중 분류** (binary가 아닌 weighted) |
| *(해당 없음)* | SWEL_m | **BSW-OS 독자** — 시맨틱 인프라 실측 |
| *(해당 없음)* | MQ | **BSW-OS 독자** — 추천 강도 분리 측정 |

## 6. KAIVI — Korea AI Visibility Index

> **"대한민국 시장에서 브랜드의 AI 가시성 종합 수준"**

### 6.1 정의

KAIVI(Korea AI Visibility Index)는 BAIR, AEPI, Per-Layer Metrics, MRI 4개 독립 원천 지표를 종합하여 워크스페이스의 **국가 수준 AI 가시성**을 단일 수치로 산출하는 최상위 지수입니다.

- **범위:** 0~100
- **단위:** 점
- **주기:** 주간 또는 월간

### 6.2 측정 방법론

#### 6.2.1 KAIVI V2 공식 (4원 복합)

$$\text{KAIVI} = 0.30 \times \text{BAIR}_{avg} + 0.30 \times \text{AEPI}_{avg} + 0.25 \times \text{PerLayer}_{avg} + 0.15 \times \text{MRI}$$

$$0 \leq \text{KAIVI} \leq 100$$

#### 6.2.2 4개 원천 상세

| 원천 | 가중치 | 측정 관점 | 데이터 소스 |
|------|--------|----------|-----------|
| **BAIR** | **30%** | Output-side — AI가 브랜드를 어떻게 말하는가 | 프로브 응답 분석 |
| **AEPI** | **30%** | Input-side — AI가 브랜드 정보를 얼마나 아는가 | 엔티티 반영 스냅샷 |
| **Per-Layer** | **25%** | 전략 성과 — 실전 전투력 | IRI·BDR·CWR 벤치마크 |
| **MRI** | **15%** | 데이터 성숙도 — 운영진실 검증 상태 | 운영진실 approved/verified 비율 |

#### 6.2.3 가중치 설계 근거

```
BAIR  (30%)    ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓  ← "AI가 뭐라고 하는가" (Output)
AEPI  (30%)    ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓  ← "AI가 뭘 알고 있는가" (Input)
PerLayer (25%) ▓▓▓▓▓▓▓▓▓▓▓▓▓    ← "실전에서 이기고 있는가" (Performance)
MRI   (15%)    ▓▓▓▓▓▓▓▓          ← "데이터가 준비됐는가" (Readiness)
```

> **BAIR와 AEPI에 동일 가중치(30%)를 부여한 근거:**
> - BAIR는 **결과(Output)** 관점 — AI 응답에서 실제로 어떻게 표현되는가
> - AEPI는 **원인(Input)** 관점 — AI가 엔티티 정보를 얼마나 정확히 보유하는가
> - 결과와 원인 모두 동등하게 중요하며, 각각 독립적 측정 경로를 가짐

> **PerLayer가 25%인 근거:** IRI·BDR·CWR은 BAIR의 세부 분해가 아닌 **레이어별 독립 측정**이므로 별도 가중치 부여. OPP는 "기회"이므로 현재 가시성 종합에서는 제외.

> **MRI가 15%인 근거:** 데이터 성숙도는 가시성의 **선행 지표**(leading indicator)이지 가시성 자체가 아니므로 보조 가중치.

#### 6.2.4 KAIVI Breakdown 투명 분해

KAIVI 산출 시 항상 원천 지표별 분해(breakdown)를 함께 제공합니다.

```json
{
  "kaivi": 58.72,
  "breakdown": {
    "bairAvg": 62.40,
    "aepiAvg": 55.30,
    "perLayerAvg": 61.00,
    "mri": 48.50
  },
  "industriesEvaluated": 3,
  "measuredAt": "2026-06-29T10:30:00Z"
}
```

> **해석:** "KAIVI 58.72점. AEPI(55.3)와 MRI(48.5)가 BAIR(62.4)에 비해 낮으므로, 엔티티 반영도와 데이터 자산 구축을 우선 강화해야 합니다."

### 6.3 과학성 분석

| 평가 기준 | 등급 | 근거 |
|----------|------|------|
| **다원 복합** | ⭐⭐⭐⭐⭐ | 단일 지표 의존 없이 4개 독립 원천 종합 — 단일 장애점 없음 |
| **원천 독립성** | ⭐⭐⭐⭐⭐ | BAIR·AEPI·PerLayer·MRI 각각 독립 데이터 소스 & 독립 측정 경로 |
| **분해 가능성** | ⭐⭐⭐⭐⭐ | 항상 breakdown 제공 — 종합 점수의 원인 추적 가능 |
| **범위 보장** | ⭐⭐⭐⭐⭐ | 가중 합산 + clamp — 수학적 보장 |
| **모의값 제거** | ⭐⭐⭐⭐⭐ | 모든 기본값(default) 제거 — 데이터 없으면 0 (추정값 미주입) |

### 6.4 활용 가치

> **경영진 요약:** "우리 KAIVI는 58점으로, AI 가시성이 중간 수준입니다. 가장 큰 약점은 엔티티 반영도(AEPI=55)와 데이터 자산(MRI=48)이며, 이를 개선하면 KAIVI를 70점대로 끌어올릴 수 있습니다."

- **시계열 추적:** 월간 KAIVI 추이로 AI 가시성 투자의 ROI 측정
- **업종 간 비교:** 동일 매크로 카테고리 내 기업 간 KAIVI 벤치마크
- **투자자 커뮤니케이션:** 디지털 자산의 AI-readiness를 단일 지표로 전달

```
Phase 1: 프로브 설계          Phase 2: 실측          Phase 3: 지표 산출
                                                    
┌──────────────────┐    ┌──────────────────┐    ┌──────────────────┐

│ × Fair Probe Set │    │ ChatGPT, Gemini  │    │ → AEPI, BAIR,   │
│   (편향 방지)    │    │ Perplexity 등    │    │   Per-Layer, ... │
└──────────────────┘    └──────────────────┘    └──────────────────┘
```

### 8.2 편향 방지 메커니즘

| 메커니즘 | 설명 |
|---------|------|
| **Fair Probe Set Builder** | L2(경쟁)/L7(브랜드) 질문을 브랜드 수 × K회 반복 생성하여 모든 브랜드가 동일 횟수 노출 |
| **Multi-Engine 동시 측정** | ChatGPT, Gemini 등 다수 엔진의 응답을 병렬 수집하여 특정 엔진 편향 상쇄 |
| **한국어/영어 이중 매칭** | 한국어는 substring, 영어는 word boundary 매칭으로 각 언어 특성에 맞는 정밀도 확보 |
| **Fuzzy Korean Match** | 한국어 정규화(조사 제거, 띄어쓰기 통합) 후 매칭으로 false negative 최소화 |

### 8.3 통계적 신뢰성 보장

| 요소 | 구현 |
|------|------|
| **최소 표본 크기** | 프로브 런 5개 미만 시 `isEstimated=true` 플래그 |
| **이동 평균** | OCR 등 변동성 높은 지표는 최근 5개 스냅샷 평균 사용 |
| **Confidence Penalty** | 표본 < 5: confidence=0.60, ≥ 5: confidence=0.95 |
| **Volatility Penalty** | 과거 10개 스냅샷의 ARS 표준편차 기반 변동성 감산 |

### 8.4 정보 검색 이론과의 정합성

| BSW-OS 개념 | IR/NLP 이론적 기반 |
|------------|-------------------|
| AEPI 7차원 가중치 | **Weighted Relevance Scoring** — 차원별 적합성 가중 합산 (Robertson, 1977) |
| 반영도 4단계 | **Graded Relevance** — NDCG(Normalized DCG)에서 사용하는 다단계 적합성 |
| Attractor Stability (M7) | **Jaccard Similarity** — 집합 간 유사도 측정 (Jaccard, 1912) |
| Drift Score (M8) | **Cosine Distance** — 벡터 공간 모델의 표준 거리 척도 |
| Variance (M12) | **Bernoulli Variance** — 이항 분포의 분산 p(1-p) |
| 업종별 가중치 | **Query-Dependent Relevance** — 질의 유형에 따른 차별적 가중 (Fuhr, 2008) |
