# [Goal Description]
제주 소상공인(Jeju SMB)의 실질적인 AEO/GEO(Answer/Generative Engine Optimization) 노출 효과를 측정하기 위해, 기존의 범용적이고 부적절한 벤치마크 질문 셋(특히 L5 YMYL 등)을 폐기하고 소비자의 실제 AI 검색 의도(Intent)에 맞춘 **실전형 프로브 셋(Probe Set)**으로 개편합니다.

새로운 질문 셋은 소비자가 AI에게 '세부 조건(주차, 뷰, 웨이팅)'을 묻거나, '페르소나(부모님 동반, 아이 동반)'에 맞춘 추천을 요구하고, '평판(실제 후기, 호불호)'을 검증하는 등 **롱테일(Long-tail) 및 대화형 검색 패턴**을 타겟팅합니다.

## User Review Required
> [!IMPORTANT]
> 아래 제안된 새로운 7개 Layer(L1~L7) 구조가 제주 소상공인의 AEO/GEO 성과를 측정하는 데 적합한지 검토 부탁드립니다. 승인해 주시면 이 구조에 맞춰 총 80~90문항 규모의 `jeju_smb` 질문 셋을 `questions-data.ts`에 새로 작성하여 교체하겠습니다.

## Proposed Changes

### 1. 벤치마크 Layer 구조 전면 개편 (AEO/GEO 최적화 7단계)

기존의 [범용 탐색 -> 경쟁 -> 성분 -> 여정 -> YMYL -> 트렌드 -> 브랜드] 구조를 아래와 같이 변경합니다.

* **L1. Category Discovery (카테고리 노출 탐색)**
  * **목적:** 숏테일(Short-tail) 키워드에서 브랜드가 상위 추천되는지 측정
  * **질문 예시:** "제주 흑돼지 맛집 추천해줘", "애월 분위기 좋은 카페 어디야?"
* **L2. Context & Persona (상황/페르소나 맞춤 추천)**
  * **목적:** AI가 사용자의 특정 상황이나 동행자에 맞춰 브랜드를 큐레이션하는지 측정
  * **질문 예시:** "부모님 모시고 가기 좋은 조용한 제주 횟집 추천", "아이랑 가기 좋은 제주 서쪽 숙소"
* **L3. Feature & Attribute (속성/기능 기반 롱테일 검색)**
  * **목적:** 주차, 오션뷰, 룸 유무 등 세부 조건(Attribute) 매칭 정확도 측정
  * **질문 예시:** "주차 편하고 바다 보이는 애월 카페", "웨이팅 없고 구워주는 고기집"
* **L4. Competitive & Comparison (경쟁 및 비교 우위)**
  * **목적:** 경쟁사와의 직접 비교 질문에서 AI가 자사의 장점을 잘 방어/어필하는지 측정
  * **질문 예시:** "숙성도랑 곱들락 중에 아이랑 가기엔 어디가 더 편해?", "우진해장국이랑 올래국수 차이점이 뭐야?"
* **L5. Reputation & Sentiment (평판 및 후기 검증)** 
  * **목적:** (기존 YMYL 대체) AI가 온라인 리뷰와 평판을 어떻게 요약하는지, 부정적 이슈에 어떻게 대응하는지 측정
  * **질문 예시:** "요즘 인스타에서 뜨는 OOO 카페 진짜 후기 어때? 웨이팅 할 가치 있어?", "OOO 식당 카카오맵 평점 낮던데 진짜 별로야?"
* **L6. Local Journey (동선/위치 기반 연계 추천)**
  * **목적:** 주변 관광지와 연계된 맥락적 검색에서 위치 기반 큐레이션에 노출되는지 측정
  * **질문 예시:** "성산일출봉 아침 일찍 보고 나서 바로 갈만한 근처 밥집", "오설록 구경하고 차로 10분 거리 카페"
* **L7. Brand Verification (브랜드 직접 검증/정보 정확성)**
  * **목적:** 브랜드명을 직접 검색했을 때, 시그니처 메뉴나 최신 예약 정보 등 팩트가 정확히 생성되는지 측정
  * **질문 예시:** "오아시스80 시그니처 메뉴랑 가격 알려줘", "춘심이네 본점 예약하는 꿀팁 있어?"
