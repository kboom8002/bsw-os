## 2. Quick 진단 (무료)

### 2.1 진단 실행 방법

1. `https://bsw-os.vercel.app/ko/site-audit` 접속
2. 웹사이트 URL 입력 (예: `https://laneige.com`)
3. **진단 시작** 버튼 클릭
4. ~5-10초 후 결과 대시보드 표시

### 2.2 Quick 진단에서 수행되는 분석

| 단계 | 분석 내용 | 소요 시간 |
|------|---------|----------|
| 크롤링 | 최대 5페이지 HTML 수집, robots.txt, sitemap, llms.txt 확인 | ~2초 |
| L1 기술 인프라 | AI 봇 접근성, HTTPS, TTFB, 사이트맵, 렌더링 모드 | ~1초 |
| L2 구조화 시맨틱 | Schema.org 8종 검사, OG 태그, 메타 태그 감사 | ~1초 |
| L3 콘텐츠 시맨틱 | E-E-A-T, Answer-First 문체, 신선도, 링크 토폴로지 | ~1초 |
| 엔티티 추출 | HTML 기반 12종 엔티티 분류 (AI API 미사용) | ~1초 |
| 갭 분석 | 4-사분면 갭 + 처방전 생성 | <1초 |
| 업종 포지셔닝 | 업종 자동 감지 → 벤치마크 프로필 대조 (프로필 존재 시) | <1초 |

> **참고:** Quick 진단은 AI API를 호출하지 않습니다. 순수 HTML 파싱만으로 분석하므로 `auditMode = 'estimated'`(추정값)로 표시됩니다.

### 2.3 업종 자동 감지

시스템은 URL과 추출된 엔티티 텍스트를 분석하여 업종을 자동으로 감지합니다.

**감지 로직:** 13개 업종별 한국어/영어 키워드 세트와 매칭하여, 2개 이상 키워드가 일치하는 업종을 선택합니다.

**지원 업종:** skincare, wedding_studio, beauty, clinic, restaurant, real_estate, education, travel, pet, convenience_retail, fashion_ecommerce, it_software, food_beverage

## 3. 진단 결과 대시보드 — 10개 탭 가이드

진단이 완료되면 10개 탭으로 구성된 대시보드가 표시됩니다.

### 탭 1: 📊 진단 개요 (모든 티어)

전체 진단 결과를 한눈에 보여주는 요약 패널입니다.

**표시 내용:**
- **4-Layer 점수 카드:** L1 기술(100점), L2 스키마(100점), L3 콘텐츠(100점), 반영도(100점)
- **AEPI 점수:** AI 엔진 가시성 지수 (0-100) — 기술 수정자 × E-E-A-T 수정자 적용
- **ERR 레이더 차트:** 7차원 엔티티 반영률 (factoid, procedural, comparative, authority, schema_org, topical_cluster, local_geo)
- **4-사분면 갭 매트릭스:** GREEN(유지)/YELLOW(개선)/RED(생성)/WHITE(기회)
- **핵심 이슈 목록:** L1/L2/L3에서 추출된 치명적 이슈
- **시계열 트렌드:** 최근 90일 AEPI 추이 (데이터 있을 때)

### 탭 2: 🖥️ 기술 인프라 (모든 티어)

Layer 1 기술 인프라 감사 결과를 상세히 보여줍니다.

**표시 내용:**
- **종합 점수 (원형 게이지):** techInfraScore (0-100)
- **SSL/HTTPS 상태:** 인증서 유효성, 만료일
- **TTFB 속도:** 밀리초 + 등급 (fast/moderate/slow)
- **렌더링 모드:** SSR / CSR / Hybrid 감지
- **AI 봇 접근 매트릭스:** 7개 봇(GPTBot, OAI-SearchBot, Google-Extended, PerplexityBot, Anthropic-AI, Bingbot, *)의 Allow/Disallow 상태
- **사이트맵:** URL 수, 90일 신선도 점수
- **llms.txt:** 존재 여부 및 내용
- **Canonical 태그 일관성:** 페이지별 canonical 설정 비율

### 탭 3: 📋 구조화 시맨틱 (Tier 1+)

Layer 2 Schema.org 및 메타데이터 감사 결과입니다.

**표시 내용:**
- **종합 점수 (원형 게이지):** schemaQualityScore (0-100)
- **스키마 타입 체크리스트:** Organization, LocalBusiness, FAQPage, HowTo, Product, Article, BreadcrumbList, AggregateRating — 각각 존재 여부 + 완성도(%) + 진행 바
- **Organization 신뢰 데이터:** 로고, 연락처, sameAs 프로필 목록
- **OpenGraph 완성도:** og:title, og:description, og:image, og:type, og:url 각각 체크
- **메타 태그 감사 테이블:** 페이지별 title/description/canonical 상태
- **이슈 목록:** 스키마 관련 개선 사항

### 탭 4: ✍️ 콘텐츠 시맨틱 (Tier 1+)

Layer 3 콘텐츠 품질 및 시맨틱 분석 결과입니다.

**표시 내용:**
- **종합 점수 (원형 게이지):** contentSemanticScore (0-100)
- **E-E-A-T 4축 차트:** Experience, Expertise, Authoritativeness, Trustworthiness 각 0-100
- **5개 하위 탭:**
  - **Answer-First 문체:** 페이지별 첫 문장 길이, 정의 포함 여부, 질문 제목 여부, 점수
  - **콘텐츠 신선도:** 평균 나이(일), 90일 이내 비율, 최신/가장 오래된 페이지
  - **내부 링크 토폴로지:** 총 내부 링크, 페이지당 평균, 허브 페이지, 고아 페이지
  - **멀티모달 자산:** 이미지 alt 텍스트 비율/품질, 임베드 비디오 수
  - **인용/출처망:** 외부 링크 수, 권위 도메인 비율(.gov/.edu), nofollow 비율

### 탭 5: 💊 최적화 처방전 (Tier 1+)

갭 분석 기반 구체적 개선 처방전 목록입니다.

**표시 내용:**
- 우선순위순 정렬된 처방전 테이블
- 각 처방전: 우선순위 점수, 처방 유형(색상 배지), 대상 엔티티, 상세 설명, 예상 AEPI 영향(+pt)
- **처방 유형:** 스키마 추가, 제목 개선, EEAT 시그널 추가, 콘텐츠 생성, 내부링크 개선, FAQ 마크업, 메타 개선, 기회 콘텐츠

### 탭 6: 📦 지식 자산 (Tier 1+)

웹사이트에서 추출된 모든 엔티티(지식 자산) 목록입니다.

**표시 내용:**
- 타입별 필터 탭 (전체/factoid/procedural/comparative/authority/schema_org/topical_cluster/local_geo + 수량)
- 스크롤 가능 테이블: 엔티티명, 타입, 완성도 점수, EEAT 강도, 스키마 지원(✓/✗), 출처 URL

### 탭 7: 📊 업종 포지셔닝 (Tier 1+)

업종 내 상대적 위치를 22개 지표에서 시각화합니다.

> **전제 조건:** 해당 업종의 벤치마크 프로필이 사전에 생성되어 있어야 합니다. (관리자가 배치 감사를 실행한 후)

**표시 내용:**
- **전체 등급:** S/A/B/C/D/F (백분위 기반)
- **업종 레이더 차트:** 6축 비교 (내 사이트 vs 업종 평균 vs 업종 Top P75)
- **22개 메트릭 백분위 바:** L1/L2 2열 + L3 전폭, P25/P50/P75 마커 표시
- **강점 목록:** 백분위 75 이상인 지표 (초록색 하이라이트)
- **약점 목록:** 백분위 50 미만인 지표 (빨간색 하이라이트)
- **Gap-to-Best 테이블:** 현재값 vs 우수 평균, 격차, Blueprint 권고사항

### 탭 8: 🎯 개선 전략 (Tier 1+)

업종 Blueprint 기반 맞춤형 개선 전략을 우선순위로 제시합니다.

**표시 내용:**
- **현재 등급 → 목표 등급:** (예: C → B)
- **경영진 요약:** 한 줄 전략 방향
- **⚡ Quick Wins:** 쉬운 구현 + 중간~높은 영향 (최대 5개)
- **📈 우선순위 전략 테이블:** 순위, 전략명, 카테고리, Gap, 난이도, 영향도
- **전략 카드 (펼치기/접기):**
  - 순위 + 카테고리 배지 + 난이도/영향도 태그
  - 3열 수치 비교: 현재값 / 업종 평균 / 업종 Top
  - 근거 설명
  - 구체적 실행 방법 (액션 아이템 리스트)
- **🏗️ 장기 투자:** 어렵지만 영향 큰 전략 (최대 3개)

### 탭 9: 🤖 AI 페르소나 (Tier 2+)

AI 엔진이 브랜드를 어떤 톤과 포지셔닝으로 표현하는지 분석합니다.

**표시 내용:**
- **톤 벡터 5축 비교:** 따뜻함, 격식, 자신감, 전문성, 공감 (의도 vs 관측)
- **델타 알림:** 15% 이상 괴리 시 경고
- **정렬 점수:** 페르소나 정렬(0-100), 바이브 정렬(0-100)
- **시장 포지셔닝:** 카테고리, 경쟁 프레임, 감정 가치, 추천 강도

### 탭 10: 🧪 8D 시뮬레이션 (Tier 3)

8차원 파라메트릭 페르소나 시뮬레이션 결과입니다.

**표시 내용:**
- **전체 강도 게이지:** B2C vs B2B 채널 강도 비교
- **Vibe 허용 대역 드리프트:** 축별 OVER/UNDER 표시
- **충실도 점수:** Baseline vs Conditioned 비교 + 델타
- **7개 차원 바:** Persona Fidelity, Vibe Alignment, Mode Switch 등
- **Floor Risk 평가:** SAFE/CAUTION/VULNERABLE/CRITICAL 등급
- **PersonaSpec OS YAML:** 원본 스펙 표시

## 4. Full 감사 (유료)

### 4.1 Full 감사 실행 방법

1. Quick 진단 완료 후, 대시보드 상단의 **"Full Audit 업그레이드"** 배너 클릭
2. **Lite (~3분)** 또는 **Pro (~8분)** 선택
3. 결제/이메일 입력 후 감사 시작
4. 진행 상태 페이지로 자동 이동 (`/site-audit/progress/{sessionId}`)
5. 14단계 진행률이 실시간으로 표시됨
6. 완료 시 결과 페이지로 자동 리다이렉트 (`/site-audit/results/{sessionId}`)

### 4.2 Full 감사 14단계 상세

| # | 단계 | 설명 |
|---|------|------|
| 0 | Quick 기준선 | 안전 폴백용 기본 데이터 수집 |
| 1 | 웹사이트 크롤링 | 최대 20페이지 심층 크롤링 |
| 2 | L1 기술 인프라 | robots.txt, SSL, TTFB, 사이트맵, llms.txt |
| 3 | L2 구조화 시맨틱 | Schema.org 8종, OG, 메타 태그 |
| 4 | AI 엔티티 추출 | Gemini Flash로 최대 10페이지 분석 |
| 5 | 지식 그래프 구축 | 엔티티 관계 + 의미 추론 |
| 6 | 앤서 카드 역설계 | AI 답변 카드 구조 역공학 |
| 7 | 프로브 질의 생성 | 비교/심층/상황별 질의 생성 |
| 8 | QIS 교차 매핑 | 업종 QIS와 사이트 QIS 교차 매핑 |
| 9 | L3 콘텐츠 시맨틱 | EEAT, Answer-First, 신선도 등 9개 분석 |
| 10 | AI 반영도 실측 | 최대 5개 프로브로 멀티 엔진 실측 |
| 11 | AEPI 점수 산출 | 업종별 가중치 적용 복합 점수 |
| 12 | 페르소나 분석 | Tier1→v1, Tier1.5+→v2/v3 페르소나 |
| 13 | 갭 분석 | 4-사분면 갭 + 14종 처방전 |
| 14 | 시계열 추적 | 90일 AEPI 추이 기록 |

> **안전 메커니즘:** 각 단계는 독립적으로 try/catch 처리됩니다. 한 단계가 실패해도 Quick Audit 데이터로 graceful fallback하여 나머지 단계가 계속 진행됩니다.

### 4.3 감사 결과 보기

Full 감사 결과는 고유 URL로 영구 보관됩니다:
```
https://bsw-os.vercel.app/ko/site-audit/results/{sessionId}
```

이 URL에서 10개 탭 전체에 접근할 수 있으며, `auditMode`가 `measured`(실측)로 표시됩니다.

## 5. 업종 벤치마크 관리 (관리자)

### 5.2 업종 벤치마크 생성 프로세스

#### Step 1: 세부 업종 선택

화면 상단의 업종 선택기에서 분석할 세부 업종을 선택합니다.
- **녹색 점(●)** 표시: 시드 데이터가 있는 업종 (현재: 스킨케어)
- **회색** 표시: 레퍼런스 사이트 큐레이션이 필요한 업종

#### Step 2: 감사 모드 선택

| 모드 | 설명 | 소요 시간 | AI API |
|------|------|----------|--------|
| ⚡ Quick (HTML-only) | HTML 파싱 기반 3-Layer 진단 | 5-10초/사이트 | 불필요 |
| 🚀 Full Audit | AI 반영도 포함 정밀 감사 | 60-120초/사이트 | 필요 |

#### Step 3: 배치 감사 실행

**[스킨케어 배치 감사 실행]** 버튼을 클릭하면:
1. 선택된 업종의 모든 레퍼런스 사이트를 순차적으로 감사
2. 각 사이트에서 22개 메트릭의 SiteAuditSnapshot 추출
3. 진행 바에 실시간 진행률 표시
4. 완료 시:
   - **IndustryBenchmarkProfile** (백분위 분포, 우수 패턴, 공통 함정) 자동 생성
   - **IndustryBlueprint** (표준 설계안 4개 섹션) 자동 생성
   - DB에 upsert (동일 업종 재실행 시 갱신)

#### Step 4: 결과 확인

감사 완료 후 화면에 3개 섹션이 표시됩니다:

1. **감사 결과 테이블:** 사이트별 L1/L2/L3 점수, E-E-A-T, AI봇 접근 점수 (색상 코딩)
2. **표준 설계안 (Blueprint):** 4개 섹션(기술 인프라, 스키마, 콘텐츠, 디자인)별 목표 점수, 현재 업종 평균, 권고사항
3. **우수 사이트 공통 패턴:** 우수/미흡 사이트 간 지표 격차와 actionable insight

### 5.3 기존 데이터 불러오기

**[기존 데이터 불러오기]** 버튼으로 이전에 생성된 벤치마크 프로필과 Blueprint를 DB에서 불러올 수 있습니다.

## 6. 업종 포지셔닝 & 개선 전략 활용

### 6.1 활성화 조건

업종 포지셔닝과 개선 전략 탭이 활성화되려면:
1. ✅ 해당 업종의 **벤치마크 프로필**이 사전에 생성되어 있어야 함 (관리자가 배치 감사 실행)
2. ✅ 진단 대상 사이트의 **업종이 자동 감지**되어야 함 (또는 수동 입력)
3. ✅ 사용자 **티어가 Tier 1 이상**이어야 함

### 6.2 업종 포지셔닝 읽는 법

#### 전체 등급 (S~F)

| 등급 | 백분위 | 의미 |
|------|--------|------|
| **S** | ≥90 | 업종 최상위 10% — 리더십 유지에 집중 |
| **A** | ≥75 | 상위 25% — 미세 조정으로 S급 진입 가능 |
| **B** | ≥60 | 상위권 — 핵심 약점 1-2개 해결 시 A급 |
| **C** | ≥40 | 평균 수준 — 체계적 개선 필요 |
| **D** | ≥25 | 하위권 — 기본기부터 점검 필요 |
| **F** | <25 | 하위 25% — 긴급 개선 필요 |

#### 레이더 차트

6축 오버레이 차트로 내 사이트(실선)와 업종 평균(점선), 업종 Top P75(점선)을 비교합니다.
- 안쪽으로 함몰된 축 = 약점 (우선 개선 대상)
- 바깥으로 돌출된 축 = 강점 (유지/홍보 대상)

#### 백분위 바

각 메트릭의 업종 내 위치를 시각적으로 보여줍니다.
- P25/P50/P75 마커로 업종 분포를 한눈에 파악
- 초록(상위) → 파랑(평균) → 빨강(하위) 색상 코딩

### 6.3 개선 전략 활용법

#### Quick Wins 우선 실행

Quick Wins는 **구현이 쉽고 효과가 큰** 항목입니다. 카드를 펼쳐서:
1. **현재값 vs 업종 평균 vs 업종 Top** 수치 비교 확인
2. **근거** 섹션에서 왜 이 전략이 중요한지 이해
3. **실행 방법** 리스트의 액션 아이템을 순서대로 실행

#### 우선순위 전략 테이블 활용

테이블 행을 클릭하면 해당 전략의 상세 카드가 펼쳐집니다.
- **Gap 수치:** 현재값과 업종 최상위 간 격차 (클수록 개선 여지 큼)
- **난이도:** 쉬움(초록)/보통(노랑)/어려움(빨강)
- **영향도:** 🔥🔥🔥(높음) / 🔥🔥(중간) / 🔥(낮음)

> **권장 순서:** Quick Wins → 우선순위 전략 상위 3개 → 장기 투자 순으로 실행

## 7. 레퍼런스 사이트 큐레이션 (관리자)

### 7.1 레퍼런스 사이트 선정 기준

각 세부 업종당 **최소 9개 사이트** (우수 3 + 평균 3 + 미흡 3)를 큐레이션해야 정확한 벤치마크가 생성됩니다.

#### 우수 사이트 (Excellent) 선정 기준
- 업종 내 AEO/SEO에서 인정받는 리딩 브랜드
- Schema.org 마크업이 풍부
- E-E-A-T 시그널이 강함
- AI 검색 결과에서 높은 가시성

#### 평균 사이트 (Average) 선정 기준
- 업종 내 중간 규모 브랜드
- 기본적인 SEO는 구현했으나 AEO 최적화는 미흡
- 일부 스키마 마크업 존재

#### 미흡 사이트 (Poor) 선정 기준
- AEO/SEO 최적화가 거의 없는 사이트
- Schema.org 미구현 또는 불완전
- robots.txt에서 AI 봇 차단
- 콘텐츠가 오래되었거나 구조화되지 않음

### 7.2 레퍼런스 사이트 추가/삭제

#### 추가
업종 벤치마크 관리 페이지 하단의 **레퍼런스 사이트 추가** 폼에서:
1. **URL:** 사이트 전체 주소 (https:// 포함)
2. **브랜드명:** 한글 또는 영문 브랜드명
3. **티어:** 우수(Excellent) / 평균(Average) / 미흡(Poor)
4. **큐레이터 메모:** 선정 사유 (선택)
5. **[사이트 추가]** 버튼 클릭

#### 삭제
각 사이트 카드 우측 상단의 🗑️ 아이콘을 클릭합니다.

### 7.3 추가 업종 확장

현재 스킨케어만 시드 데이터가 제공됩니다. 다른 업종을 추가하려면:

1. 업종 벤치마크 관리 페이지에서 원하는 업종 선택
2. 위의 기준에 따라 우수/평균/미흡 사이트를 각 3개 이상 추가
3. 배치 감사 실행
4. 이후 해당 업종의 브랜드가 Quick 진단을 받으면 자동으로 포지셔닝이 표시됨

> **코드 레벨 확장:** `lib/industry/reference-sites-registry.ts`의 `REFERENCE_SITES_REGISTRY` 맵에 직접 시드 데이터를 추가할 수도 있습니다.

## 10. FAQ

### Q: Quick 진단의 추정값(estimated)과 Full 감사의 실측값(measured)의 차이는?
**A:** Quick 진단은 HTML 파싱만으로 AI 반영도를 **추정**합니다 (엔티티 완성도 기반 간접 추론). Full 감사는 실제 AI 엔진에 프로브 질의를 보내 응답에서 브랜드가 **실제로 어떻게 표현되는지 직접 측정**합니다.

### Q: 업종 벤치마크를 자주 갱신해야 하나요?
**A:** 분기별(3개월) 재실행을 권장합니다. 업종 내 사이트들은 지속적으로 SEO/AEO를 개선하므로, 오래된 프로필은 정확도가 떨어질 수 있습니다.

### Q: 다른 업종 시드 데이터는 어떻게 추가하나요?
**A:** 두 가지 방법이 있습니다:
1. **UI:** 업종 벤치마크 관리 페이지의 레퍼런스 사이트 추가 폼 사용
2. **코드:** `lib/industry/reference-sites-registry.ts`에 직접 추가

### Q: Quick 감사에서도 업종 포지셔닝이 표시되나요?
**A:** 네! 해당 업종의 벤치마크 프로필이 이미 DB에 존재하면, Quick 감사에서도 자동으로 포지셔닝을 계산합니다. 단, Free 티어에서는 잠금 처리되어 Tier 1 이상에서만 볼 수 있습니다.

### Q: AEPI 점수에서 기술 수정자(techModifier)와 EEAT 수정자(eeatModifier)의 역할은?
**A:** 두 수정자는 각각 0.8~1.0 범위의 승수입니다.
- `techModifier = 0.8 + 0.2 × (techScore / 100)` → 기술 인프라가 완벽(100점)이면 1.0, 0점이면 0.8
- `eeatModifier = 0.8 + 0.2 × (eeatScore / 100)` → E-E-A-T가 완벽이면 1.0, 0점이면 0.8
- 즉, 기술/EEAT가 아무리 나빠도 AEPI가 최대 20%까지만 감소하는 안전 장치입니다.

### Q: 레퍼런스 사이트를 삭제하면 벤치마크 프로필에 즉시 반영되나요?
**A:** 아닙니다. 레퍼런스 사이트 삭제 후 **배치 감사를 재실행**해야 새로운 프로필이 생성됩니다. 기존 프로필은 upsert로 덮어쓰기됩니다.

### Q: 한 업종에 레퍼런스 사이트가 3개 미만이면?
**A:** 배치 감사는 실행 가능하지만, 통계적 유의성이 낮아 백분위 분포의 정확도가 떨어집니다. **우수/평균/미흡 각 3개 이상(총 9개 이상)을 권장합니다.**
