# 특허 명세서 — AI 검색 최적화 기반 웹사이트 자동 구축·운영 시스템

> **기술 분류:** G06Q 30/00 (전자상거래), G06F 16/958 (웹 정보검색), G06N 20/00 (기계학습)

## Ⅰ. 발명의 개요

### 1. 발명의 명칭

**"업종 역설계 지능 기반 AI 검색 최적화 웹사이트의 콜드스타트 자동 구축 및 자기 진화 시스템"**

(영문: "System and Method for Cold-Start Auto-Construction and Self-Evolving AI-Search-Optimized Websites Using Industry Reverse-Engineering Intelligence")

### 2. 기술 분야

본 발명은 AI 검색 엔진(ChatGPT, Perplexity, Gemini 등)의 응답에서 브랜드가 인용·추천되는 가시성(AEO: AI Engine Optimization)을 극대화하기 위해, (1) 업종 전체의 우수/평균/열등 사이트를 배치 역설계하여 표준 설계안(Blueprint)을 도출하고, (2) 최소 입력만으로 상위 25%(A-) 수준의 웹사이트를 자동 구축하며, (3) 질문 자산의 양방향 교환 및 예측 콘텐츠 파이프라인을 통해 구축 후 자기 진화하는 통합 시스템에 관한 것이다.

### 3. 종래 기술의 문제점

| # | 종래 기술 | 한계 |
|---|---------|------|
| 1 | 기존 SEO 진단 도구 (Screaming Frog, Ahrefs 등) | 기존 사이트의 기술적 결함만 진단. AI 검색 시맨틱 가시성 미측정. 콜드스타트 지원 불가. |
| 2 | 웹사이트 빌더 (Wix, Squarespace 등) | 업종 지능 없이 범용 템플릿만 제공. AI 봇 최적화(llms.txt, Schema.org) 미흡. 구축 후 자동 성장 없음. |
| 3 | AI 콘텐츠 생성 도구 (ChatGPT, Jasper 등) | 업종 벤치마크·Expected Layer·YMYL 안전 기준 미반영. 일회성 생성, 폐루프 성장 없음. |
| 4 | 업종 분석 도구 | 개별 경쟁사 분석만 수행. 업종 전체의 티어별(우수/평균/열등) 배치 역설계 후 통계적 표준 도출 없음. |
| 5 | 질문 분석 도구 (AnswerThePublic 등) | 검색 볼륨만 제공. 질문의 비즈니스 가치(QVS), 출현 예측, AI 검색 커버리지, Expected Layer 없음. |

**핵심 미해결 과제:**
> 디지털 자산이 전무한(Near-Zero) 소상공인이 최소 입력만으로 업종 상위 수준의 AI 검색 최적화 웹사이트를 즉시 구축하고, 이후 자동으로 성장시키는 통합 기술이 부재함.

### 4. 발명의 목적

본 발명은 다음을 목적으로 한다:

1. 업종 전체를 티어별로 배치 역설계하여 통계적 표준 설계안(Blueprint)을 자동 도출하는 방법
2. 최소 입력(브랜드명 + 3문장 + 감성 벡터)만으로 상기 Blueprint를 기반으로 A- 수준 웹사이트를 자동 구축하는 방법
3. 질문 자산의 양방향 교환과 예측 콘텐츠 자동 생성을 통해 구축 후 자기 진화하는 방법
4. 상기 자기 진화의 효과를 정량 측정하여 시스템 자체를 지속 학습시키는 방법

## Ⅱ. 특허성 높은 핵심 발명 요소 (12개 독립항)

### ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

### 발명 1: 업종 티어별 배치 역설계 및 통계적 표준 설계안 자동 도출 방법

**특허성 근거:** 개별 사이트 분석이 아닌, 업종 내 복수 사이트를 **우수(excellent)/평균(average)/열등(poor) 3개 티어**로 분류하여 배치 감사하고, 티어 간 통계적 차이에서 **Industry Blueprint**를 자동 도출하는 방법은 선행 기술에 존재하지 않음.

**해자성:** 누적된 업종 벤치마크 데이터가 쌓일수록 Blueprint 정확도가 높아지는 **데이터 네트워크 효과**. 경쟁자가 동일 수준의 Blueprint를 만들려면 동등한 사이트 감사 데이터 축적 필요.

#### 청구항 1

업종(industry)에 속하는 복수의 웹사이트를 티어(tier)별로 분류하여 배치 역설계하고, 상기 티어 간 통계적 분포 차이로부터 업종 표준 설계안을 자동 도출하는 방법으로서,

(a) 업종 및 세부 업종(sub-industry)에 대하여 우수(excellent), 평균(average), 열등(poor) 3개 티어로 분류된 복수의 레퍼런스 사이트를 등록하는 단계;

(b) 상기 각 레퍼런스 사이트에 대하여, 기술 인프라 계층(L1), 구조화 시맨틱 계층(L2), 콘텐츠 시맨틱 계층(L3)의 3계층 역설계 감사를 병렬 수행하여 각각의 감사 스냅샷(snapshot)을 생성하는 단계;
- 상기 L1은 렌더링 모드, 응답 시간, HTTPS, AI 크롤러 허용 상태, 사이트맵 유효성, llms.txt 존재 여부를 포함하고,
- 상기 L2는 Schema.org JSON-LD의 종류·수량·밀도·유효성, 메타 태그 완성도를 포함하고,
- 상기 L3는 EEAT(Experience, Expertise, Authoritativeness, Trustworthiness) 시그널, Answer-First 문체 비율, 콘텐츠 신선도, 내부 링크 밀도, 멀티모달 활용도를 포함함;

(c) 상기 감사 스냅샷을 세부 업종별로 집계하여, 각 메트릭의 P25·P50·P75 백분위 분포를 산출하고, 티어별 평균 점수 및 패턴을 추출하는 단계;

(d) 상기 티어별 통계적 분포 차이로부터, 기술 인프라 표준, 구조화 시맨틱 표준, 콘텐츠 전략, 디자인 패턴, 목표 점수를 포함하는 업종 표준 설계안(Industry Blueprint)을 자동 생성하는 단계;

(e) 상기 우수 티어 사이트에서만 공통적으로 발견되는 패턴을 **우수 패턴(excellent_patterns)**으로, 열등 티어에서만 공통적으로 발견되는 패턴을 **공통 함정(common_pitfalls)**으로 분류하여 설계안에 포함하는 단계

를 포함하는 업종 표준 설계안 자동 도출 방법.

### 발명 2: 업종 대리 감사(Industry Proxy Audit) 기반 콜드스타트 웹사이트 설계 방법

**특허성 근거:** 기존 역설계는 반드시 대상 사이트가 존재해야 함. 본 발명은 대상 사이트 없이(Near-Zero) **업종 Blueprint만으로** 웹사이트 설계 명세를 생성하는 "대리 감사(Proxy Audit)" 개념을 최초로 도입함.

**해자성:** "사이트 없이도 진단 받을 수 있다"는 고객 경험 자체가 진입 장벽. 경쟁자가 모방하려면 발명 1의 업종 Blueprint 데이터가 전제 조건.

#### 청구항 2

웹사이트가 존재하지 않는 브랜드에 대하여, 업종 표준 설계안을 대리 감사 데이터로 활용하여 웹사이트 설계 명세를 생성하는 방법으로서,

(a) 사용자로부터 업종 식별자(industry_type) 및 최소 브랜드 정보를 수신하는 단계;

(b) 상기 업종 식별자에 대응하는 업종 표준 설계안(Industry Blueprint) 및 업종 벤치마크 프로필(Benchmark Profile)을 청구항 1의 방법으로 사전 생성된 데이터베이스에서 로드하는 단계;

(c) 상기 Blueprint의 목표 점수(P75 기준)를 웹사이트의 각 품질 메트릭의 목표치로 설정하고, 상기 우수 패턴을 설계 요구사항으로, 공통 함정을 설계 회피사항으로 변환하여 웹사이트 설계 명세를 생성하는 단계;

(d) 상기 설계 명세를 웹사이트 자동 구축 시스템에 전달하여, 실제 자사 진단 데이터 없이 업종 상위 25% 수준의 웹사이트를 구축하는 단계

를 포함하는 콜드스타트 웹사이트 설계 방법.

### 발명 3: 다차원 감성 벡터와 업종 역설계 데이터의 코사인 유사도 기반 아키타입 자동 매칭 방법

**특허성 근거:** (1) 사용자 감성 5문항을 7차원 벡터로 변환하고, (2) 업종 역설계 진단 결과를 동일 7차원 벡터로 투영한 후, (3) 코사인 유사도로 최적 아키타입을 매칭하여 디자인+GNB+콘텐츠+품질기준을 일괄 결정하는 융합 방법은 선행 기술 없음.

**해자성:** 11+ 아키타입 × 업종별 Vec7D 매핑 데이터 × Elo 캘리브레이션 학습 결과가 쌓일수록 매칭 정확도 증가. 후발 주자가 동등한 아키타입 데이터셋을 구축하기 어려움.

#### 청구항 3

웹사이트 구축 시 디자인·구조·콘텐츠를 일괄 결정하기 위한 아키타입 자동 매칭 방법으로서,

(a) 사용자에게 복수의 양극성(bipolar) 감성 질문을 제시하고, 각 응답으로부터 N차원(N≥5) 감성 벡터(Vec-ND)를 산출하는 단계;
- 상기 감성 벡터의 각 차원은 온기(warmth), 에너지(energy), 세련됨(polish), 진정성(authentic), 전통성(heritage), 미래지향(futuristic), 유쾌함(playful) 중 2개 이상을 포함함;

(b) 업종 역설계 감사 결과의 품질 메트릭(EEAT, 기술 인프라, 콘텐츠 시맨틱 점수)을 상기 동일한 N차원 벡터 공간으로 투영(projection)하는 단계;

(c) 상기 사용자 감성 벡터와 업종 투영 벡터를 가중 합산하여 최종 목표 벡터를 산출하는 단계;

(d) 복수의 사전 정의된 아키타입 각각의 목표 벡터와 상기 최종 목표 벡터 간 코사인 유사도를 계산하여, 유사도가 최대인 아키타입을 선택하는 단계;

(e) 상기 선택된 아키타입에 포함된 디자인 토큰 번들, GNB/IA 구조, 콘텐츠 자산 매트릭스, 품질 기준 프로필, 질문 자산 시드를 웹사이트 구축에 일괄 적용하는 단계

를 포함하는 아키타입 자동 매칭 방법.

### 발명 4: 5계층 심리 가중치 기반 웹페이지 섹션 자동 배치 방법

**특허성 근거:** 역설계 진단 결과의 약점 영역을 5단계 소비자 심리 모델(Attention→Trust→Value→Proof→Action)에 매핑하여, 약점 심리 계층의 가중치를 높이고 해당 계층의 UI 섹션을 자동으로 강화 배치하는 방법은 선행 기술 없음.

**해자성:** 심리 모델과 역설계 진단의 교차 매핑이 핵심 알고리즘. 표면적 UI 배치는 모방 가능하나, "진단 결과에 따라 심리 계층별 가중치가 동적으로 변하는" 로직은 알고리즘 특허로 보호 가능.

#### 청구항 4

웹사이트의 홈페이지 섹션 배치를 자동 결정하는 방법으로서,

(a) 웹사이트의 품질 감사 결과 또는 업종 표준 설계안으로부터, 주목(attention), 신뢰(trust), 가치(value), 증거(proof), 전환(action)의 5개 소비자 심리 계층 각각에 대한 가중치를 산출하는 단계;
- 상기 가중치는, 해당 심리 계층에 대응하는 품질 메트릭이 기준치보다 낮을수록 높게 설정됨;

(b) 사전 정의된 복수의 UI 섹션 타입 각각에 부여된 심리 계층 태그를 참조하여, 상기 가중치가 높은 심리 계층에 속하는 섹션 타입을 우선 선택하는 단계;

(c) 상기 선택된 섹션들을 심리 퍼널 순서(주목→신뢰→가치→증거→전환)에 따라 자동 배치하는 단계

를 포함하는 웹페이지 섹션 자동 배치 방법.

---

### 발명 5: 업종 허브 플랫폼 기반 3-모드 콘텐츠 자동 공급 및 Blueprint 품질 보장 방법

**특허성 근거:** (1) 업종 허브에서 큐레이션된 콘텐츠를 3개 모드(즉시/커스텀/검토)로 분류하여 자동 공급하되, (2) 공급 대상이 역설계 갭 분석 결과에 매칭된 콘텐츠를 우선하고, (3) 업종 기대 레이어(Expected Layer)로 품질을 보장하는 융합 방법은 선행 기술 없음.

**해자성:** 업종 허브의 큐레이션된 콘텐츠 풀이 쌓일수록 매칭률 증가. 콘텐츠 자산 자체가 모방 불가 해자.

#### 청구항 5

콜드스타트 브랜드에 업종별 콘텐츠를 자동 공급하는 방법으로서,

(a) 업종 허브 플랫폼에 업종별 콘텐츠 풀(content pool)을 저장하되, 각 콘텐츠에 즉시 발행(direct), 커스텀 템플릿(custom_template), 검토 대기(draft)의 3개 공급 모드 중 하나를 부여하는 단계;

(b) 브랜드의 콘텐츠 갭 분석 결과 — 역설계 감사에서 도출된 부재 콘텐츠 목록 — 또는 업종 표준 설계안의 필수 콘텐츠 매트릭스를 수신하는 단계;

(c) 상기 콘텐츠 갭과 콘텐츠 풀을 매칭하여, 갭에 대응하는 콘텐츠를 식별하고, 상기 공급 모드에 따라:
- direct: 즉시 활성 상태(active)로 발행하고,
- custom_template: 브랜드 컨텍스트를 기반으로 AI가 맞춤 반영한 후 활성 발행하고,
- draft: 브랜드 담당자 검토 대기 상태로 전달하는 단계;

(d) 상기 매칭되지 않은 잔여 갭에 대해서는, 업종 기대 레이어(Expected Layer)의 5계층 기준(필수 포함, 강력 권장, 권장, 주의, 금지)을 AI 콘텐츠 생성 프롬프트에 주입하여 신규 콘텐츠를 생성하는 단계

를 포함하는 콘텐츠 자동 공급 방법.

### 발명 6: 질문 자산의 양방향 교환 및 예측 정확도 자기 강화 방법 (QIS Closed Loop)

**특허성 근거:** (1) 질문 예측 엔진이 예측 질문을 허브에 Push, (2) 허브가 1차 데이터(커뮤니티 Q&A, 실거래가, 검증 리뷰)를 Pull 제공, (3) 허브가 예측 출현 여부를 Feedback, (4) 이 피드백으로 예측 가중치를 자동 재캘리브레이션하는 **4단계 양방향 폐루프**는 선행 기술 없음.

**해자성:** 양방향 데이터 교환이 누적될수록 QVS(Question Value Score) 정확도가 향상되는 **자기 강화 피드백 루프**. 시간이 지날수록 경쟁자와의 예측 정확도 격차 확대.

#### 청구항 6

질문 자산의 양방향 교환 및 예측 정확도를 자기 강화하는 방법으로서,

(a) 질문 예측 엔진이, AI 검색 엔진의 질의 패턴을 분석하여 미래 출현 가능성이 높은 예측 질문을 생성하고, 각 예측 질문에 신뢰도(confidence), 예상 검색량, AI 커버리지 수준, 비즈니스 가치 점수(QVS)를 부여하는 단계;

(b) 상기 예측 질문 중 신뢰도가 기준치 이상인 것을 업종 허브 플랫폼에 푸시(push)하는 단계;

(c) 상기 업종 허브 플랫폼에서 운영되는 커뮤니티 질의, 검증된 리뷰, 실거래 데이터, 감정 패턴을 포함하는 1차 데이터를 수집하여 상기 질문 예측 엔진에 풀(pull) 공급하는 단계;

(d) 상기 풀 공급된 1차 데이터의 실제 전환율(conversion_rate), 실제 객단가(average_transaction), 실제 질문 빈도(question_frequency)를 상기 QVS 산출식에 반영하여 QVS를 재계산하는 단계;

(e) 상기 업종 허브에서 상기 예측 질문의 실제 출현 여부를 관측하고, 출현 결과(emerged, emerged_at, actual_frequency)를 질문 예측 엔진에 피드백하는 단계;

(f) 상기 피드백을 기반으로 예측 모델의 신호 가중치를 자동 재캘리브레이션하여, 이후 예측 주기의 정확도를 향상시키는 단계

를 포함하는 질문 자산 양방향 교환 및 자기 강화 방법.

### 발명 7: 예측 질문 기반 콘텐츠 자동 생성 및 시한부 리뷰 강제 방법

**특허성 근거:** 예측 질문을 (1) 비즈니스 가치(QVS) 정렬, (2) 기대 레이어 준수 AI 드래프트, (3) 스키마 자동 래핑, (4) **24시간 리뷰 마감 후 자동 비활성화**의 4단계로 처리하는 파이프라인은 선행 기술 없음. 특히 시한부 자동 비활성화는 콘텐츠 품질과 신선도를 강제하는 독창적 메커니즘.

**해자성:** 파이프라인 자체의 알고리즘 보호 + Expected Layer 데이터의 업종별 축적.

#### 청구항 7

예측 질문으로부터 콘텐츠를 자동 생성하고, 시한부 리뷰를 강제하는 방법으로서,

(a) 질문 예측 엔진이 생성한 예측 질문 중, 신뢰도(confidence) ≥ 기준치이고 AI 검색 커버리지가 부재(none) 또는 희박(sparse)인 질문을 필터링하는 단계;

(b) 상기 필터링된 질문을 비즈니스 가치 점수(QVS) 순으로 정렬하여 상위 N건을 선택하는 단계;

(c) 상기 선택된 질문 각각에 대하여, 업종 기대 레이어(Expected Layer)의 필수 포함(must_include) 및 금지(must_not_do) 항목을 AI 생성 프롬프트에 주입하여 콘텐츠 드래프트를 자동 생성하는 단계;

(d) 상기 질문의 의도 유형(intent)에 따라 Schema.org 마크업을 자동 결정하여 래핑하는 단계;
- 추천(recommendation) → FAQPage, 비교(comparison) → Article, 행동(action_seeking) → HowTo, 지역(local_intent) → LocalBusiness를 포함함;

(e) 상기 생성된 콘텐츠에 리뷰 마감 시한(review_deadline)을 설정하고, 상기 마감 시한 내에 브랜드 담당자의 승인이 없는 경우 상기 콘텐츠를 자동 비활성화하는 단계

를 포함하는 예측 질문 기반 콘텐츠 자동 생성 방법.

### 발명 8: 역설계 갭 분석과 질문 예측의 교차 부스팅 방법

**특허성 근거:** (1) 사이트 역설계 갭(콘텐츠 부재 영역)과 (2) 질문 예측 결과를 교차 검증하여, 양쪽 모두에서 식별된 토픽의 우선순위를 부스트하는 방법은 선행 기술 없음. "역설계"와 "예측"이라는 이질적 분석 축의 교차가 핵심 신규성.

**해자성:** 역설계 데이터와 QIS 예측 데이터를 동시에 보유해야만 교차 부스팅 가능. 단일 기능 경쟁자는 구현 불가.

#### 청구항 8

웹사이트 역설계 갭 분석과 질문 예측을 교차하여 콘텐츠 우선순위를 결정하는 방법으로서,

(a) 웹사이트의 3계층 역설계 감사 결과로부터, 업종 표준 대비 부재하는 콘텐츠 엔티티를 갭(gap)으로 식별하고, 각 갭을 적색(부재), 황색(불충분), 백색(블루오션)으로 분류하는 단계;

(b) 질문 예측 엔진이 동일 업종에 대하여 미래 출현 가능성이 높은 예측 질문을 생성하는 단계;

(c) 상기 갭 목록과 예측 질문 목록을 어휘적 유사도 및 의미적 유사도(임베딩 코사인 유사도)로 교차 매칭하는 단계;

(d) 상기 교차 매칭에서 양쪽 모두에 존재하는 토픽(갭 ∩ 예측)에 대하여 우선순위 점수를 가산(boost)하는 단계;
- 상기 가산은 예측 질문의 신뢰도(confidence)에 기준 값을 더하는 방식을 포함함;

(e) 상기 갭에만 존재하는 토픽은 질문 예측 엔진에 신규 시드로 등록하고, 예측에만 존재하는 토픽은 갭 분석에 미래 기회로 추가하여, 양방향으로 분석 범위를 확장하는 단계

를 포함하는 교차 부스팅 방법.

### 발명 9: Blueprint 가이드 AI 콘텐츠 생성 방법

**특허성 근거:** AI 콘텐츠 생성 시, (1) 업종 벤치마크의 우수 패턴을 양성 프롬프트로, (2) 공통 함정을 음성 프롬프트로, (3) 기대 레이어의 5계층 기준을 품질 가드레일로, (4) 브랜드 차별화 포인트를 개인화 요소로 동시 주입하는 4중 프롬프트 엔지니어링 방법은 선행 기술 없음.

**해자성:** 업종별로 축적된 excellent_patterns + common_pitfalls + Expected Layer 데이터가 프롬프트 품질을 결정. 데이터 없이 프롬프트만 모방해도 동등 품질 달성 불가.

#### 청구항 9

업종 표준 설계안을 AI 콘텐츠 생성 프롬프트에 주입하여 고품질 콘텐츠를 생성하는 방법으로서,

(a) 업종 표준 설계안(Blueprint)으로부터 우수 패턴(excellent_patterns) 및 공통 함정(common_pitfalls)을 추출하는 단계;

(b) 업종 기대 레이어(Expected Layer)로부터 필수 포함, 강력 권장, 권장, 주의, 금지의 5계층 기준을 추출하는 단계;

(c) 브랜드의 최소 입력(창업 스토리, 경쟁 우위, 고객 걱정사항)으로부터 차별화 요소를 추출하는 단계;

(d) 상기 (a)의 우수 패턴을 양성 지시사항으로, 공통 함정을 음성 지시사항으로, (b)의 기대 레이어를 품질 가드레일로, (c)의 차별화 요소를 개인화 지시사항으로 포함하는 복합 프롬프트를 구성하는 단계;

(e) 상기 복합 프롬프트를 대형 언어 모델(LLM)에 입력하여 콘텐츠를 생성하되, 생성된 콘텐츠가 상기 기대 레이어의 금지 항목을 위반하지 않음을 검증하는 단계

를 포함하는 Blueprint 가이드 AI 콘텐츠 생성 방법.

### 발명 10: AI 검색 엔진 가시성 통합 지표(AEPI) 및 3중 교차 검증 방법

**특허성 근거:** 기술 인프라(L1), 구조화 시맨틱(L2), 콘텐츠 시맨틱(L3)의 3계층 점수를 가중 합산하여 단일 AI 검색 가시성 지표(AEPI)를 산출하고, 이를 GEO Score 및 BAIR Score와 3중 교차 검증하는 방법은 선행 기술 없음.

**해자성:** AEPI 지표 자체의 표준화 지위 선점. 업계에서 "AEO 진단 = AEPI"로 인식되면 경쟁자가 대안 지표를 만들어도 전환 비용 발생.

#### 청구항 10

웹사이트의 AI 검색 엔진 가시성을 정량화하는 통합 지표를 산출하고, 복수의 독립 지표와 교차 검증하는 방법으로서,

(a) 웹사이트에 대하여 기술 인프라 계층(L1), 구조화 시맨틱 계층(L2), 콘텐츠 시맨틱 계층(L3) 각각의 점수를 산출하는 단계;

(b) 상기 L1, L2, L3 점수를 사전 정의된 가중치로 합산하여 단일 통합 지표(AEPI: AEO Performance Index, 0-100)를 산출하는 단계;

(c) 동일 웹사이트에 대하여, AI 검색 엔진에 프로브 질문을 입력하여 해당 웹사이트의 인용 여부·순위를 관측하는 GEO 점수를 독립적으로 산출하는 단계;

(d) 동일 웹사이트에 대하여, AI Answer Share(AAS), Official Citation Rate(OCR), Brand Semantic Fidelity(BSF)의 가중 합산으로 BAIR 점수를 독립적으로 산출하는 단계;

(e) 상기 AEPI, GEO, BAIR 3개 지표를 교차 비교하여, 불일치가 기준치를 초과하는 경우 재진단을 자동 트리거하는 단계

를 포함하는 AI 검색 가시성 통합 지표 및 교차 검증 방법.

### 발명 11: 아키타입 효과의 Elo 레이팅 자기 학습 방법

**특허성 근거:** 웹사이트 구축에 적용된 아키타입의 사후 효과(AEPI 변화량)를 Elo 레이팅 시스템에 피드백하여, 업종별 최적 아키타입을 자기 학습하는 방법은 선행 기술 없음. 체스·게임에서 사용되는 Elo 시스템을 웹사이트 디자인 선택의 효과 측정에 적용하는 것은 신규 응용.

**해자성:** 아키타입 적용 → 사후 AEPI 측정 → Elo 업데이트의 루프가 시간과 사용자 수에 비례하여 데이터가 축적됨. 경쟁자가 동등한 Elo 데이터를 쌓으려면 동등한 시간과 사용자 기반 필요.

#### 청구항 11

웹사이트 구축에 적용된 디자인 아키타입의 효과를 자기 학습하는 방법으로서,

(a) 브랜드의 웹사이트 구축 시, 복수의 아키타입 후보 중 선택된 하나의 아키타입을 적용하고, 적용 시점의 품질 지표(AEPI_before)를 기록하는 단계;

(b) 사전 정의된 기간 경과 후, 상기 웹사이트의 품질 지표(AEPI_after)를 재측정하여, 변화량(AEPI_delta = AEPI_after - AEPI_before)을 산출하는 단계;

(c) 상기 AEPI_delta를 상기 적용된 아키타입의 Elo 레이팅에 반영하되, 동일 업종·동일 기간에 다른 아키타입이 적용된 웹사이트의 AEPI_delta와 상대 비교하여 Elo 레이팅을 갱신하는 단계;

(d) 이후 동일 업종의 신규 웹사이트 구축 시, 상기 Elo 레이팅이 높은 아키타입을 우선 추천하는 단계

를 포함하는 아키타입 효과 자기 학습 방법.

### 발명 12: 진단-구축-성장-재진단 완전 자동화 폐루프 시스템

**특허성 근거:** (1) 업종 역설계 진단 → (2) 아키타입 매칭 + Hub 콘텐츠 자동 구축 → (3) QIS+Growth 자동 성장 → (4) 재진단 + Elo 학습의 4단계가 인간 개입 없이 완전 자동으로 순환하는 폐루프 시스템은 선행 기술 없음. 각 단계의 출력이 다음 단계의 입력이 되는 연쇄적 자동화가 핵심.

**해자성:** 전체 폐루프를 구현하려면 발명 1-11의 모든 구성요소가 필요. 부분 모방으로는 폐루프 효과를 재현할 수 없는 **시스템 레벨 해자**.

#### 청구항 12

웹사이트의 AI 검색 가시성을 자동으로 구축하고 지속 개선하는 완전 자동화 폐루프 시스템으로서,

(a) 업종 역설계 모듈: 업종에 속하는 복수의 웹사이트를 티어별로 배치 역설계하여 업종 표준 설계안(Blueprint)을 자동 도출하고, 콜드스타트 브랜드에 대하여 상기 Blueprint를 대리 감사 데이터로 활용하여 설계 명세를 생성하는 모듈;

(b) 자동 구축 모듈: 상기 설계 명세 및 최소 브랜드 입력으로부터, 감성 벡터 기반 아키타입 매칭(청구항 3), 심리 계층 기반 섹션 배치(청구항 4), Hub 콘텐츠 자동 공급(청구항 5), Blueprint 가이드 AI 생성(청구항 9)을 수행하여 웹사이트를 자동 구축하는 모듈;

(c) 자동 성장 모듈: 구축 후, QIS 양방향 질문 교환(청구항 6), 예측 콘텐츠 자동 생성(청구항 7), 갭-예측 교차 부스팅(청구항 8)을 주기적으로 실행하여 콘텐츠를 자동 확장하는 모듈;

(d) 자동 재진단 모듈: 사전 정의된 주기 또는 이벤트 트리거에 따라, AEPI 재측정 및 3중 교차 검증(청구항 10)을 수행하고, AEPI 변화량을 아키타입 Elo 레이팅(청구항 11)에 피드백하는 모듈;

(e) 상기 (a)~(d)가 (a)→(b)→(c)→(d)→(a)의 순서로 인간 개입 없이 자동 순환하되, 각 단계의 출력이 다음 단계의 입력으로 사용되며, 순환 횟수가 증가할수록 상기 Blueprint, 아키타입 Elo, QVS 정확도, 콘텐츠 품질이 모두 향상되는 자기 강화 특성을 갖는 것을 특징으로 하는 완전 자동화 폐루프 시스템.
